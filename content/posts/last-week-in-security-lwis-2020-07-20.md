<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://research.checkpoint.com/2020/resolving-your-way-into-domain-admin-exploiting-a-17-year-old-bug-in-windows-dns-servers/" target="_blank">SIGRed – Resolving Your Way into Domain Admin: Exploiting a 17 Year-old Bug in Windows DNS Servers</a>. This bug is interesting because it could be triggered by simply visiting a web page by a user that is using a Windows DNS server (i.e. AD). While this is a heap exploit and no public PoC exists yet (besides <a href="https://github.com/maxpl0it/CVE-2020-1350-DoS" target="_blank">this DoS</a>), researchers likely have already weaponized this vulnerability.</li>
<li><a href="https://www.onapsis.com/recon-sap-cyber-security-vulnerability" target="_blank">Unpatched SAP Systems at High Risk for Potential Cyber Threats and Compliance Violations</a>. If by "Compliance Violations" they mean unauthenticated remote code execution then yes, "Compliance Violations." Good find by Onapsis, but I really hate when companies lean so hard on these reports for marketing, even hiding the report itself behind an email signup. This <a href="https://us-cert.cisa.gov/ncas/alerts/aa20-195a" target="_blank">CISA Alert</a> has some details. Check for vulnerability with <a href="https://github.com/chipik/SAP_RECON" target="_blank">SAP_RECON</a>.</li>
<li><a href="https://krebsonsecurity.com/2020/07/whos-behind-wednesdays-epic-twitter-hack/" target="_blank">Who’s Behind Wednesday’s Epic Twitter Hack?</a>. If you missed it, many high profile (Bill Gates, Barak Obama, Jeff Bezos, etc) twitter accounts tweeted about a Bitcoin scam last Wednesday. To me, the ability to snoop on direct messages seems more valuable than the ~$100,000 in Bitcoin the attackers received. TLDR: Twitter's internal Slack had credentials to an administration system that allowed the account access. Maybe the attackers used <a href="https://github.com/PaperMtn/slack-watchman" target="_blank">slack-watchman</a>?</li>
<li><a href="https://www.theguardian.com/technology/2020/jul/17/tiktok-says-australian-users-data-wont-be-shared-with-foreign-powers" target="_blank">TikTok says Australian users' data won't be shared with foreign powers</a>. However, TikTok's own <a href="https://www.tiktok.com/safety/resources/transparency-report?lang=en" target="_blank">transparency report</a> show that they, unsurprisingly, hand over a lot of data to governments when asked. The report makes no mention of requests from China.</li>
<li><a href="https://www.cyberscoop.com/whatsapp-nso-group-spying-continues-judge-rules/" target="_blank">WhatsApp lawsuit against NSO Group spying can proceed, judge rules</a>. This is an interesting ruling that could allow Facebook, who owns WhatsApp, to get details on NSO Groups clients. NSO Group has always walked a fine line, as they provide "exploitation as a service" vs the more legally defensible vulnerability sales.</li>
<li><a href="https://theintercept.com/2020/07/16/face-masks-facial-recognition-dhs-blueleaks/" target="_blank">Homeland Security Worries Covid-19 Masks Are Breaking Facial Recognition, Leaked Document Shows</a>. "The potential impacts that widespread use of protective masks could have on security operations that incorporate face recognition systems — such as video cameras, image processing hardware and software, and image recognition algorithms — to monitor public spaces during the ongoing Covid-19 public health emergency and in the months after the pandemic subsides." As seems to the be the case with most authoritarian issues, China is way ahead: <a href="https://www.siliconrepublic.com/enterprise/china-gait-recognition-cctv" target="_blank">Chinese authorities use gait analysis to identify citizens on CCTV</a> (2018).</li>
<li><a href="http://sandlab.cs.uchicago.edu/fawkes/" target="_blank">Image "Cloaking" for Personal Privacy</a>. This is great research that allows users to alter photos of faces to fool common AI/ML facial recognition technology while appearing almost unaltered to the human eye. I love research like this that is well presented and releases the <a href="https://github.com/Shawn-Shan/fawkes" target="_blank">code</a>. So many potentially cool research papers never release their code, so bravo to <a href="https://twitter.com/shawnshan26" target="_blank">@shawnshan26</a> and the Fawkes team.</li>
<li><a href="https://arstechnica.com/information-technology/2020/07/iran-state-hackers-caught-with-their-pants-down-in-intercepted-videos/" target="_blank">Iranian state hackers caught with their pants down in intercepted videos</a>. The take away from this article is twofold: 1. Use a password manager (unique password per site) and 2FA, and 2. Open S3 buckets affect everyone, even the hackers. This group is considered an Advanced Persistent Threat (APT)...</li>
<li><a href="https://archive.ooo/" target="_blank">DEF CON Playable CTF Archives</a>. Order of the Overflow has really done a great job with the DEF CON CTF in recent years. This site allows you to access and actually play past challenges from both the DEF CON CTF and the qualifier events.</li>
<li><a href="https://blog.zsec.uk/cve-2020-1350-honeypoc/" target="_blank">HoneyPoC: The fallout data after I trolled the Internet...</a>. This is interesting because it shows just how hungry attackers are for the latest vulnerabilities. Likely access from DPRK and GRU cyber operators (with poor OPSEC) within days of a CVE release show that even if you are not staying current on the latest CVEs, the APTs are.</li>
</ul>

<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://codewhitesec.blogspot.com/2020/07/sophos-xg-tale-of-unfortunate-re.html" target="_blank">Sophos XG - A Tale of the Unfortunate Re-engineering of an N-Day and the Lucky Find of a 0-Day</a>. You may recall the Sophos XG N-Day from early 2020, but it turns out there was more to the story. This very detailed write up by <a href="https://twitter.com/ramoliks" target="_blank">@ramoliks</a> and <a href="https://twitter.com/niph_" target="_blank">@niph_</a> shows how their research into that exploit lead them to another unauthenticated remote code execution 0day. Bravo!</li>
<li><a href="https://rastamouse.me/blog/process-injection-dinvoke/" target="_blank">Process Injection using DInvoke</a>. <a href="https://twitter.com/_RastaMouse" target="_blank">@_RastaMouse</a> demonstrates the differences between the standard P/Invoke and the new D/Invoke technique in C# and how it can be use to hide API calls from AV/EDR solutions.</li>
<li><a href="https://www.forrest-orr.net/post/masking-malicious-memory-artifacts-part-ii-insights-from-moneta" target="_blank">Masking Malicious Memory Artifacts Part II: Insights from Moneta</a>. <a href="https://twitter.com/_ForrestOrr" target="_blank">@_ForrestOrr</a> doesn't post often, but when he does you owe it to yourself to read his posts. This is another well researched and written article, which contains offensive and defensive technique gold: "Interestingly, the takeaway concept from this analysis seems to be that attempting to detect such memory is nearly impossible with IOCs alone when the malware writer understands the landscape he is operating in and takes care to camouflage his tradecraft in one of the many existing abnormalities in Windows. <a href="https://github.com/forrest-orr/moneta" target="_blank">Moneta</a> provides a useful way for attackers to identify such abnormalities and customize their dynamic code to best leverage them for stealth. Similarly, it provides a valuable way for defenders to identify/dump malware from memory and also to identify the false positives they may be interested in using to fine-tune their own memory detection algorithms." Time to blend your tools in with the "false-positives" already present in Windows.</li>
<li><a href="https://www.matteomalvica.com/blog/2020/07/15/silencing-the-edr/" target="_blank">Silencing the EDR. How to disable process, threads and image-loading detection callbacks.</a>. <a href="https://twitter.com/matteomalvica" target="_blank">@matteomalvica</a> uses the hacker's favorite GigaByte driver to disable driver signature enforcement, then loading a custom driver that allows the selective disabling of kernel callbacks that EDR uses. Example code <a href="https://github.com/fdiskyou/windows-ps-callbacks-experiments" target="_blank">here</a>. Looking for more driver exploitation for EDR bypass? Check out <a href="https://www.redcursor.com.au/blog/bypassing-lsa-protection-aka-protected-process-light-without-mimikatz-on-windows-10" target="_blank">Bypassing LSA Protection (aka Protected Process Light) without Mimikatz on Windows 10</a>. Drivers may be <a href="https://twitter.com/RedVuln/status/1284829109215006720" target="_blank">fertile ground</a> for exploit research.</li>
<li><a href="https://unit42.paloaltonetworks.com/windows-server-containers-vulnerabilities/" target="_blank">Windows Server Containers Are Open, and Here’s How You Can Break Out</a>. Symbolic Links strike again, this time allowing any code in a Windows Server Container to break out and have full control of the host. If you are running Windows Server Containers on Azure Kubernetes Service (AKS), this escape allows for lateral movement to other cluster nodes as well.</li>
<li><a href="https://labs.f-secure.com/assets/BlogFiles/2020-07-the-fake-cisco.pdf" target="_blank">The Fake Cisco</a>. This report from F-Secure explores a counterfeit Cisco device in depth and discovers an exploit that allowed signature and platform verification to pass on fake devices - until it didn't. Very cool hardware hacking. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1195/003/" target="_blank">T1195.003 Supply Chain Compromise: Compromise Hardware Supply Chain</a>]</li>
<li><a href="https://posts.specterops.io/persistent-aws-access-with-role-chain-juggling-9f96acd348e9" target="_blank">Persistent AWS access with role chain juggling</a>. "Temporary credentials" may be used to issue more "temporary credentials," making them not so temporary.</li>
<li><a href="https://www.trustedsec.com/blog/a-developers-introduction-to-beacon-object-files/" target="_blank">A Developer's Introduction to Beacon Object Files</a>. Still confused about Beacon Object Files in Cobalt Strike 4.1? Christopher Paschen writes a good overview of what they are, why they are cool, and some tips and tricks to get started using them.</li>
</ul>
</section>
<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/fireeye/capa" target="_blank">capa</a> detects capabilities in executable files. You run it against a PE file or shellcode and it tells you what it thinks the program can do. For example, it might suggest that the file is a backdoor, is capable of installing services, or relies on HTTP to communicate. Details on the <a href="https://www.fireeye.com/blog/threat-research/2020/07/capa-automatically-identify-malware-capabilities.html" target="_blank">Fireeye Blog</a>.</li>
<li><a href="https://github.com/JordanKnott/project-citadel" target="_blank">project-citadel</a> is a free &amp; open source alternative project management tool that offers basic task tracking through a Kanban board (think Trello).</li>
<li><a href="https://github.com/yeswehack/pwn-machine" target="_blank">pwn-machine</a> is a self hosting solution based on docker aiming to provide an easy to use pwning station for bughunters. This is a first release, but it could become a very cool platform to help automate the backend stuff required to find interesting bugs. More information in this <a href="https://blog.yeswehack.com/yeswerhackers/the-pwning-machine/" target="_blank">blog post</a>.</li>
<li><a href="https://labs.redyops.com/index.php/2020/07/14/mcafee-total-protection-mtp-16-0-r26-escalation-of-privilege-cve-2020-7283/" target="_blank">McAfee Total Protection (MTP) &lt; 16.0.R26 Escalation of Privilege (CVE-2020-7283)</a>. Another AV allows for local privilege escalation due to symlink mishandling and overly permissive permissions. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1068/" target="_blank">T1068 Exploitation for Privilege Escalation</a>]</li>
<li><a href="https://github.com/leechristensen/RequestAADRefreshToken/" target="_blank">RequestAADRefreshToken</a> obtains a refresh token for an Azure-AD-authenticated Windows user (i.e. the machine is joined to Azure AD and a user logs in with their Azure AD account). An attacker can then use the token to authenticate to Azure AD as that user. More info on the Specter Ops <a href="https://posts.specterops.io/requesting-azure-ad-request-tokens-on-azure-ad-joined-machines-for-browser-sso-2b0409caad30" target="_blank">blog</a>.</li>
<li><a href="https://github.com/rasta-mouse/RuralBishop" target="_blank">RuralBishop</a> is practically a carbon copy of <a href="https://github.com/FuzzySecurity/Sharp-Suite/tree/master/UrbanBishop" target="_blank">UrbanBishop</a> by b33f, but all P/Invoke calls have been replaced with D/Invoke.</li>
</ul>
