<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://www.fireeye.com/blog/threat-research/2020/12/evasive-attacker-leverages-solarwinds-supply-chain-compromises-with-sunburst-backdoor.html" target="_blank">Highly Evasive Attacker Leverages SolarWinds Supply Chain to Compromise Multiple Global Victims With SUNBURST Backdoor</a>. This is the first good example of some proper hacking we've seen in a long time. The past few years have seen reporters and cybersecurity companies slap the "Advanced Persistent Threat" (APT) label on any group capable of getting a shell in a corporate environment which has weakened the term. This analysis took a village, and it shows. Excellent technical detail that exposes the lengths this actor went to prevent detection. Last week started as a PR disaster for FireEye, but the fact they were seemingly the only affected organization to detect this activity may end up actually being a big PR win. If you only read one thing this week, it should be this technical report. Lots of good tradecraft within. For defenders, load up the <a href="https://github.com/fireeye/sunburst_countermeasures" target="_blank">sunburst_countermeasures</a> and get to hunting. Krebs has an overview of the potential scope of the compromise <a href="https://krebsonsecurity.com/2020/12/u-s-treasury-commerce-depts-hacked-through-solarwinds-compromise/" target="_blank">here</a>.</li>
<li><a href="https://www.fireeye.com/blog/threat-research/2020/12/unauthorized-access-of-fireeye-red-team-tools.html" target="_blank">Unauthorized Access of FireEye Red Team Tools</a>. The big news story of the week was the compromise of FireEye by an advanced adversary - until the SolarWinds news broke. The loss of their Red Team tools feels like a red herring; you wouldn't use your Ocean's Eleven level crew/tools to steal lock picks from an Amazon warehouse. There were no 0days, and most tools appeared to be based on open source tools. The real target is likely access to the hardware/software supply chain - FireEye has lots of network devices in interesting places - as well as the Red Team report data. Why spend a lot of time finding vulnerabilities when you can pinch the reports? BishopFox has <a href="https://labs.bishopfox.com/industry-blog/the-stolen-fireeye-red-team-tools-are-mostly-open-source" target="_blank">analysis</a> on the tools based on the <a href="https://github.com/fireeye/red_team_tool_countermeasures" target="_blank">countermeasures</a> FireEye released.</li>
<li><a href="https://www.reuters.com/article/us-usa-cyber-amazon-com-exclsuive/u-s-treasury-breached-by-hackers-backed-by-foreign-government-sources-idUSKBN28N0PG" target="_blank">Suspected Russian hackers spied on U.S. Treasury emails</a>. The software supply chain is an underrated attack vector. Compromise of a supplier and the ability to backdoor updates could give an attacker access to hundreds of high value networks in a single move. This is fallout from the SolarWinds news, and shows just how many organizations were affected.</li>
<li><a href="https://blog.centos.org/2020/12/future-is-centos-stream/" target="_blank">CentOS Project shifts focus to CentOS Stream</a>. "Shifts focus" is a polite way to say "abandons commitment to stability." CentOS 8 is now EOL 2021-12-31 instead of the previously stated 2029-05-31 - a casual 7.5 year change. <a href="https://github.com/rocky-linux/rocky" target="_blank">rocky-linux</a> looks poised to take up the "100% bug-for-bug compatible with RHEL" cause and is backed by the CentOS founder. IBM's acquisition of Red Hat is not getting off to a good start.</li>
<li><a href="https://github.com/oskarsve/ms-teams-rce" target="_blank">ms-teams-rce</a>. This is a great example of the potential dangers of Electron. An XSS bug lead to cross platform RCE. Unfortunately, Microsoft gave this the lowest severity rating and also said that Electron apps (like Teams) are out of scope. Not a good look. Simply viewing a chat could trigger the RCE. The bugs were patched in October 2020.</li>
<li><a href="https://rizin.re/" target="_blank">rizin</a> is a new fork of <a href="https://github.com/radareorg/radare2" target="_blank">radare2</a> that states its purpose is to focus on stability and inclusiveness. They also brought the team behind <a href="https://cutter.re/" target="_blank">Cutter</a> which is a nice cross-platform C++ GUI frontend. Will be interesting to see how this fork goes.</li>
<li><a href="https://blog.cloudflare.com/oblivious-dns/" target="_blank">Improving DNS Privacy with Oblivious DoH in 1.1.1.1</a>. This is a good step toward private DNS lookups. It is basically a limited and fast Tor network for DNS-over-HTTPS queries with proxies that can't see content and resolvers that only see requests from proxies. I hope it is adopted into the <cite>cloudflared</cite> stub resolver soon.</li>
<li><a href="https://blog.cloudflare.com/encrypted-client-hello/" target="_blank">Good-bye ESNI, hello ECH!</a>. As someone <a href="https://github.com/SixGenInc/Noctilucent" target="_blank">very interested in ESNI</a> this is a great progress! The new advances like the <a href="https://tools.ietf.org/html/draft-ietf-dnsop-svcb-https-02" target="_blank">HTTPS resource record type</a> and <a href="https://tools.ietf.org/html/draft-irtf-cfrg-hpke-06" target="_blank">Hybrid Public Key Encryption</a> address the issues with ESNI. ECH and Oblivious DoH give me hope for some privacy in the otherwise surveillance ridden dystopian cyberpunk future.</li>
<li><a href="https://workspaceupdates.googleblog.com/2020/12/secure-ldap-mac-os-google-login.html" target="_blank">Use Secure LDAP to log into macOS with Google credentials</a>. Google has had this ability via the Google credential provider for Windows for some time, but this is new to macOS (without using Jamf Connect). This should be useful to Google Workspace centric shops.</li>
</ul>

<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://www.n00py.io/2020/12/alternative-ways-to-pass-the-hash-pth/" target="_blank">Alternative ways to Pass the Hash (PtH)</a>. Pass-the-Hash is one of the classic lateral movement techniques for Windows environments. <a href="https://twitter.com/n00py1" target="_blank">@n00py1</a> shows some lesser known methods such as WinRM, RDP, smbclient, LDAP, Pass-the-Ticket, mount, SSH, and resource based constrained delegation.</li>
<li><a href="https://research.nccgroup.com/2020/12/10/abstract-shimmer-cve-2020-15257-host-networking-is-root-equivalent-again/" target="_blank">ABSTRACT SHIMMER (CVE-2020-15257): Host Networking is root-Equivalent, Again</a>. Containers prove to be a weak security boundary again (the UAC of linux!?). This post dives into the inner workings of containerd-shim to pull out a nice vulnerability. Exploit drops 2021-01-11.</li>
<li><a href="https://blog.raw.pm/en/state-of-the-art-of-network-pivoting-in-2019/" target="_blank">State of the art of network pivoting in 2019</a>. Don't be fooled by the title, the post is a fresh translation of a blog post from last year with lots of good content.</li>
<li><a href="https://posts.specterops.io/adventures-in-dynamic-evasion-1fe0bac57aa" target="_blank">Adventures in Dynamic Evasion</a>. This is a forward-looking post that describes how to profile and dynamically evade user mode API hooking with a two staged implant. The first stage just detects the hooks and reports back, and the C2 delivers a custom second stage that has direct syscalls in place of the hooked functions. <a href="https://github.com/matterpreter/SHAPESHIFTER" target="_blank">SHAPESHIFTER</a> is the PoC, and the Demo is <a href="https://vimeo.com/487937178" target="_blank">here</a>.</li>
<li><a href="https://blog.cobaltstrike.com/2020/12/08/a-red-teamer-plays-with-jarm/" target="_blank">Red Teamer Plays with JARM</a>. SalesForce's JARM was a interesting addition to the Blue Team toolkit, and in this post Mudge shows that the CobaltStrike JARM fingerprint is infact the Java 11 TLS stack's fingerprint. But if you're reading this, you have all your teamservers behind redirectors anyway, right?</li>
<li><a href="https://thalpius.com/2020/12/14/microsoft-on-premises-to-the-cloud-using-seamless-single-sign-on/" target="_blank">Microsoft on-premises to the cloud using Seamless Single Sign-On</a>. If you can execute code on a machine where a user is logged into Azure-AD, you can extract the ticket granting ticket of the user and the HTTP ticket granting service ticket. With these two tickets and a local policy change on your attacker VM you can load these tickets into memory with mimikatz and log into Azure or Microsoft 365 as the user! Thalpius' tool <a href="https://github.com/thalpius/Kerberos" target="_blank">Kerberos</a> now includes this as an option!</li>
<li><a href="https://luemmelsec.github.io/Kerberoasting-VS-AS-REP-Roasting/" target="_blank">AS_REP Roasting vs Kerberoasting</a> is a good post if you need a quick refresher on either AS_REP or kerberoasting with practical command examples and brief mitigations.</li>
<li><a href="https://www.fortinet.com/blog/threat-research/tutorial-arm-stack-overflow-exploit-against-setuid-root-program" target="_blank">Tutorial of ARM Stack Overflow Exploit against SETUID Root Program</a>. There are lots of stack overflow walkthroughs for x86 and even x86_64 but not as many for ARM. This is a good one that includes partial RELO (ASLR) and NX. As more IoT (and Apple computers!) move to ARM, this flavor of exploit will grow in popularity. Grab a raspberry pi and follow along!</li>
<li><a href="https://www.trustedsec.com/blog/4-free-easy-wins-that-make-red-teams-harder/" target="_blank">4 Free Easy Wins That Make Red Teams Harder</a>. This has some good quick wins for blue teams, and also should make red teams think about evolving their tradecraft to include things that aren't covered by these four easy wins.</li>
<li><a href="https://watchcom.no/nyheter/nyhetsarkiv/cisco-jabber-vulnerabilities-resurface/" target="_blank">Watchcom discovers new Cisco Jabber vulnerabilities</a>. This should sound familiar (LWiS 2020-09-07) as this company found the previous RCE bug in Cisco Jabber. It looks like the patch was a bandaid because when they did a verification audit the root cause remained and three new CVEs fell out. Another reminder that a patch is not always the death of a bug.</li>
<li><a href="https://blog.netspi.com/cve-2020-17049-kerberos-bronze-bit-attack/" target="_blank">CVE-2020-17049: Kerberos Bronze Bit Attack – Practical Exploitation</a>. This one is a bit complicated (Kerberos always is) but if you have access to a domain machine, the password hash of a service account, and that service account is allowed to perform constrained delegation to another service, you're in luck! You can use this vulnerability to authenticate to the second service as any user.</li>
</ul>

<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/nccgroup/HTTPSignatures" target="_blank">HTTPSignatures</a> is a Burp Suite extension implementing the Signing HTTP Messages <a href="https://tools.ietf.org/html/draft-ietf-httpbis-message-signatures-01" target="_blank">draft protocol</a>. This allows Burp Suite users to seamlessly test applications that require HTTP Signatures. More details in <a href="https://research.nccgroup.com/2020/12/08/tool-release-httpsignatures-a-burp-suite-extension-implementing-http-signatures/" target="_blank">this blog post</a>.</li>
<li><a href="https://bugs.chromium.org/p/project-zero/issues/detail?id=2082" target="_blank">Windows Cloud Filter Arbitrary File Creation EoP</a>. James Forshaw drops a nice little Windows LPE that abuses the Cloud Filter driver used in mounting cloud file systems like OneDrive to create arbitrary files in protected directories. A simple PoC is included, but getting it to LPE is an exercise left for the reader. The vulnerability was <a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2020-17136" target="_blank">patched</a> 2020-12-08.</li>
<li><a href="https://www.tenable.com/security/research/tra-2020-68" target="_blank">PsExec Local Privilege Escalation</a>. This one requires an attacker to create the PSEXECSVC named pipe before the PsExecSvc starts, which gives them write access to it. Since the service blindly accepts commands via that pipe, the attacker is able to execute programs as SYSTEM. Code <a href="https://github.com/tenable/poc/blob/master/Microsoft/Sysinternals/PsExecEscalate.cpp" target="_blank">here</a>.</li>
<li><a href="https://github.com/Sentinel-One/VTgrepGHIDRA" target="_blank">VTgrepGHIDRA</a> is a non official VirusTotal plugin for Ghidra which integrates VTGrep into Ghidra, facilitating the searching for similar code, strings, or sequences of bytes.</li>
<li><a href="https://github.com/SafeBreach-Labs/SirepRAT/" target="_blank">SirepRAT</a> was just updated to support Python3. This tool exploits the Sirep Test Service that’s built into the Windows IoT Core image.</li>
<li><a href="https://wadcoms.github.io/" target="_blank">WADComs</a> is an interactive cheat sheet, containing a curated list of offensive security tools and their respective commands to be used against Windows/AD environments. It's like <a href="https://gtfobins.github.io/" target="_blank">GTFOBins</a> or <a href="https://lolbas-project.github.io/#" target="_blank">LOLBAS</a> for Windows/AD exploitation.</li>
<li><a href="https://gist.github.com/X-C3LL/7bb17ecf01f59f50ad52569467af68d6" target="_blank">VBA Macro to detect EDR Hooks</a> is exactly that.</li>
<li><a href="https://github.com/asaurusrex/DoppelGate" target="_blank">DoppelGate</a> is designed to provide a method of extracting syscalls dynamically directly from on-disk ntdll. It's named DoppelGate due to the borrowing of ideas/techniques used in Process Doppelganging and the Hell's Gate technique.</li>
</ul>

<section id="new-to-me">
<h2>New to Me</h2>
<p>This section is for news, techniques, and tools that weren't released last week but are new to me. Perhaps you missed them too!</p>
<ul>
<li><a href="https://maltiverse.com/search" target="_blank">Maltiverse</a> is a (currently) free IOC search engine.</li>
<li><a href="https://github.com/ctxis/SnitchDNS" target="_blank">SnitchDNS</a> is a database driven DNS Server with a Web UI, written in Python and Twisted, that makes DNS administration easier with all configuration changed applied instantly without restarting any system services. One of its main features is the logging of all DNS queries allowing the discovery of network traffic endpoints, and it can also be used to implement canary tokens as it supports notifications via e-mail, web push, Slack, and Teams.</li>
<li><a href="https://github.com/strontic/xcyclopedia" target="_blank">xcyclopedia</a> project attempts to document all executable binaries (and eventually scripts) that reside on a typical operating system. It provides a web page to view the data as well as a machine-readable format (JSON and CSV) that can be immediately usable in other systems such as SIEMs to enrich observed executions with contextual data.</li>
<li><a href="https://github.com/TwakeApp/Twake" target="_blank">Twake</a> is a collaborative platform which improves teamwork. A self-hosted slack with video chat and collaborate document editing.</li>
</ul>
