<p>Last Week in Security is a summary of the interesting cybersecurity news, techniques, tools and exploits from the previous week. This post covers 2020-05-25 to 2020-06-01.
MITRE ATT&amp;CK techniques are in brackets where appropriate.</p>
<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://apnews.com/63c081ec091f4c1e3f438ee35243efe0" target="_blank">Israeli cyber chief: Major attack on water systems thwarted</a>. A cyber attack was reportedly detected and thwarted in real time and had it not be chlorine or other chemicals could have been mixed into the water source in the wrong proportions and resulted in a “harmful and disastrous” outcome. Israel's national cyber chief said, “Cyber winter is coming and coming even faster than I suspected. We are just seeing the beginning.”</li>
<li><a href="https://www.synacktiv.com/posts/exploit/return-of-the-ios-sandbox-escape-lightspeeds-back-in-the-race.html" target="_blank">Return of the iOS sandbox escape: lightspeed's back in the race!!</a> Synacktiv describes the old-but-new XNU bug that was re-introduced in iOS 13 and was the basis for last week's iOS 0day based jailbreak. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1068/" target="_blank">T1068 Exploitation for Privilege Escalation</a>]</li>
<li><a href="https://bhavukjain.com/blog/2020/05/30/zeroday-signin-with-apple/" target="_blank">Zero-day in Sign in with Apple</a> Speaking of Apple's mistakes, this one is quite bad. When requesting a JSON web token during the "Sign in with Apple" process, swapping the email for another user would return a valid JWT that can authenticate to the 3rd party service. This is the equivalent to editing a cookie from "user" to "admin" and getting the admin page. How this got through internal testing and into production is beyond me. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1078/" target="_blank">T1078 Valid Accounts</a>]</li>
<li><a href="https://eprint.iacr.org/2020/615.pdf" target="_blank">LadderLeak: Breaking ECDSA With Less Than One Bit Of Nonce Leakage</a> describes how a novel side-channel vulnerability can lead to leaking the most significant bit of the nonce and eventually breaking ECDSA, including NIST curves like P-192 (P-256 is vulnerable in theory). Curve25519 contains a countermeasure against this attack, but, "its exact efficacy against side-channel attacks in [this] context is not entirely clear."</li>
<li><a href="https://labs.f-secure.com/blog/releasing-the-captcha-cracken/" target="_blank">Releasing the CAPTCHA Cracken</a>. Despite the name there is no tool release, but F-Secure Labs shows off its ability to automate the CAPTCHA in use with the Office 265 Outlook Web Application, allowing its red team to attempt password stuffing. If you solely rely on CAPTCHA to stop bots/automated tools, it's time to rethink your strategy. Demo <a href="https://labs.f-secure.com/assets/Uploads/captcha-craken.webm" target="_blank">here</a>.</li>
</ul>
</section>
<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://x-c3ll.github.io/posts/GetEnvironmentVariable-Process-Injection/" target="_blank">GetEnvironmentVariable as an alternative to WriteProcessMemory in process injections</a> is a new (probably?) way to write to a suspended processes memory while avoiding the highly suspect WriteProcessMemory. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1055/" target="_blank">T1055 Process Injection</a>]</li>
<li><a href="https://blog.trailofbits.com/2020/05/27/verifying-windows-binaries-without-windows/" target="_blank">Verifying Windows binaries, without Windows</a> Trail of Bits opensoruces <a href="https://github.com/trailofbits/uthenticode" target="_blank">uthenticode</a> a small cross-platform library for verifying <a href="https://docs.microsoft.com/en-us/windows-hardware/drivers/install/authenticode" target="_blank">Authenticode</a> digital signatures.</li>
<li><a href="https://www.zerodayinitiative.com/blog/2020/5/27/mindshare-how-to-just-emulate-it-with-qemu" target="_blank">MindShaRE: How to “Just Emulate It With QEMU”</a> gives good resources to emulate embedded firmware with QEMU. "Emulating IoT Malware and Firmware with Docker+QEMU" presented last Friday at the virtual IoT village was more in-depth (<a href="https://youtu.be/N0EYsO0VxZo" target="_blank">here</a> is the same talk from AirGap 2020).</li>
<li><a href="https://redteams.fyi/" target="_blank">Red Team Maturity Model</a> is a resource for building and improving your internal Red Team. Apply the Bruce Lee principle here: “Absorb what is useful, discard what is useless, and add what is specifically your own.”</li>
<li><a href="http://jackson-t.ca/registry-data-persistence.html" target="_blank">Covert Data Persistence with Windows Registry Keys</a> details a very cool trick for storing data in the Windows Registry covertly using class attributes. I wouldn't be surprised if this technique is used by advanced malware soon, so use in in your advanced adversary emulation tools now. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1112/" target="_blank">T1112 Modify Registry</a>]</li>
<li><a href="https://medium.com/@curtbraz/these-arent-the-phish-you-re-looking-for-7374c3986af5" target="_blank">These Aren’t the Phish You’re Looking For</a> shows the power of blacklists for extending the life of your phishing site. A must read before any future phishing engagement. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1192/" target="_blank">T1192 Spearphishing Link</a>][<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1193/" target="_blank">T1193 Spearphishing Attachment</a>]</li>
<li><a href="https://research.nccgroup.com/2020/05/28/exploring-macos-calendar-alerts-part-2-exfiltrating-data-cve-2020-3882/" target="_blank">Exploring macOS Calendar Alerts: Part 2 – Exfiltrating data (CVE-2020-3882)</a> is a neat exploit that allows semi-arbitrary (calendar is sandboxed) file extraction from a victim machine if they click on the invite. Patched in macOS 10.15.5 (May 2020). [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1193/" target="_blank">T1193 Spearphishing Attachment</a>]</li>
<li><a href="https://shells.systems/automate-octopus-c2-redteam-infrastructure-deployment/" target="_blank">Automate Octopus C2 RedTeam Infrastructure Deployment</a> show the development of a script to automate the deployment of <a href="https://github.com/mhaskar/Octopus" target="_blank">Octopus</a> C2 infrastructure via digital ocean. If you don't have infrastructure automation yet, this could be a good starting point. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1090/" target="_blank">T1090 Connection Proxy</a>]</li>
<li><a href="https://pberba.github.io/security/2020/05/28/lastpass-phishing/" target="_blank">Bypassing LastPass’s “Advanced” YubiKey MFA: A MITM Phishing Attack</a> is not an attack on U2F but rather LastPass's one time password. It includes a great description of how this this possible and how U2F prevents it. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1192/" target="_blank">T1192 Spearphishing Link</a>]</li>
</ul>

<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/projectdiscovery/httpx" target="_blank">httpx</a> is a fast and multi-purpose HTTP toolkit allow to run multiple probers using retryablehttp library, it is designed to maintain the result reliability with increased threads.</li>
<li><a href="https://github.com/GhostPack/Seatbelt" target="_blank">Seatbelt</a>, while technically not a new tool got a <a href="https://github.com/GhostPack/Seatbelt/blob/master/CHANGELOG.md" target="_blank">huge rewrite</a>. Seatbelt is a C# project that performs a number of security oriented host-survey "safety checks" relevant from both offensive and defensive security perspectives. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1082/" target="_blank">T1082 System Information Discovery</a>]</li>
<li><a href="https://github.com/odedshimon/BruteShark" target="_blank">BruteShark</a> is a Network Forensic Analysis Tool (NFAT) that performs deep processing and inspection of network traffic (mainly PCAP files). It includes: password extracting, building a network map, reconstruct TCP sessions, extract hashes of encrypted passwords and even convert them to a Hashcat format in order to perform an offline Brute Force attack. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1040/" target="_blank">T1040 Network Sniffing</a>]</li>
<li><a href="https://github.com/ChrisTheCoolHut/angry_gadget" target="_blank">angry_gadget</a> - This tool is written in python and uses <a href="https://github.com/angr/angr" target="_blank">angr</a> to test constraints for gadgets executing execve('/bin/sh', NULL, NULL). If you've run out gadgets to try from OneGadget, Angry Gadget gives a lot more with complicated constraints to try!</li>
<li><a href="https://github.com/jwillyamz/ezEmu" target="_blank">ezEmu</a> enables users to test adversary behaviors via various execution techniques. Sort of like an "offensive framework for blue teamers", ezEmu does not have any networking/C2 capabilities and rather focuses on creating local test telemetry. Use this (or <a href="https://github.com/redcanaryco/atomic-red-team" target="_blank">atomic-red-team</a>) to test your SEIM, EDR, or SOC response.</li>
<li><a href="https://github.com/FortyNorthSecurity/EXCELntDonut" target="_blank">EXCELntDonut</a> is a XLM (Excel 4.0) macro generator. Start with C# source code (DLL or EXE) and end with a XLM (Excel 4.0) macro that will execute your code in memory. XLM (Excel 4.0) macros can be saved in .XLS files. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1193/" target="_blank">T1193 Spearphishing Attachment</a>]</li>
<li><a href="https://github.com/EgeBalci/sgn" target="_blank">sgn</a> Shikata ga nai encoder ported into go with several improvements including 64 bit support, smaller decoder stub, no visible loop condition, decoder stub obfuscation, and safe register option. Build this into your in house packer for better obfuscation. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1066/" target="_blank">T1066 Indicator Removal from Tools</a>]</li>
<li><a href="https://github.com/golem445/Corporate_Masks" target="_blank">Corporate_Masks</a> are 8-14 character Hashcat masks based on analysis of 1.5 million NTLM hashes cracked while pentesting. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1078/" target="_blank">T1078 Valid Accounts</a>]</li>
<li><a href="https://github.com/RythmStick/AMSITrigger" target="_blank">AMSITrigger</a> will identify all of the malicious strings in a powershell file, by repeatedly making calls to AMSI using AMSIScanBuffer, line by line. On receiving an AMSI_RESULT_DETECTED response code, the line will then be scrutinised to identify the individual triggers. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1066/" target="_blank">T1066 Indicator Removal from Tools</a>]</li>
</ul>
