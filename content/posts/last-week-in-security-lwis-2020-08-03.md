<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://www.justice.gov/usao-ndca/press-release/file/1300121/download" target="_blank">United States of America v. Nima Fazeli</a>. Some really bad OPSEC on display from the teen that hacked Twitter a few weeks ago. The voice phishing combined with the 2FA bypass was very well executed, but the same cannot be said for post-exploitation actions.</li>
<li><a href="https://9to5mac.com/2020/08/01/new-unpatchable-exploit-allegedly-found-on-apples-secure-enclave-chip-heres-what-it-could-mean/" target="_blank">New 'unpatchable' exploit allegedly found on Apple’s Secure Enclave chip, here’s what it could mean</a>. Physical access is required, and it was patched on newer devices (A12/A13 based devices) but still an interesting development for iOS security.</li>
<li><a href="https://eclypsium.com/2020/07/29/theres-a-hole-in-the-boot/" target="_blank">There's a hole in the boot</a>. Another week, another named and logo'd vulnerability. This one takes advantage of the fact that the <cite>YY_FATAL_ERROR()</cite> function of the Grub bootloader simply prints a message and then returns to the calling function, instead of actually being a fatal error that halts loading. This allows a heap buffer overflow in the config parser if an attacker can control the grub.cfg file. This one will be very hard to effectively mitigate, as blocking vulnerable versions of Grub will break existing deployments, but without blocking vulnerable versions downgrade attacks would be effective against patched machines. Update Grub and then update the revocation list to close this hole.</li>
<li><a href="https://www.f-secure.com/en/consulting/events/attack-detection-fundamentals-workshops" target="_blank">Attack Detection Fundamentals</a>. F-Secure puts out great free workshops for red and blue teamers.</li>
</ul>

<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://medium.com/@metnew/exploiting-popular-macos-apps-with-a-single-terminal-file-f6c2efdfedaa" target="_blank">Exploiting popular macOS apps with a single “.terminal” file.</a>. The quarantine attribute is not correctly set by third party applications with file transfer capabilities (i.e. Slack) for .terminal files, which can lead to single click compromise of macOS endpoints.</li>
<li><a href="https://medium.com/@mattshockl/cve-2020-9934-bypassing-the-os-x-transparency-consent-and-control-tcc-framework-for-4e14806f1de8" target="_blank">CVE-2020–9934: Bypassing the macOS Transparency, Consent, and Control (TCC) Framework for unauthorized access to sensitive user data</a>. Setting the $HOME variable in launchctl allowed Matt Shockley to modify the TCC database without disabling system integirty protection and give himself every entitlement without prompting the end user. This is a great find, and the post includes a simple script to acomplish this. If you perfer a compiled Swift binary, that is <a href="https://github.com/mattshockl/CVE-2020-9934" target="_blank">available</a> as well.</li>
<li><a href="https://posts.specterops.io/sparkling-payloads-a2bd017095c" target="_blank">Sparkling Payloads</a>. GateKeeper on macOS can be a challenge for getting payloads to run in certain cases (i.e. they are downloaded by a browser by a user). This post shows how to use a well established update framework to get benign code past Apple's review, and "update" it to a malicious payload after delivery.</li>
<li><a href="https://br-sn.github.io/Removing-Kernel-Callbacks-Using-Signed-Drivers/" target="_blank">Removing Kernel Callbacks Using Signed Drivers</a>. Kernel exploits on Windows have been featured a few times in LWiS (2020-07-20, 2020-05-11), and this post is another good example of the power of kernel level execution on Windows. Blue teams should be closely monitoring driver loads on all endpoints by this point, but if they aren't use the techniques presented in this post to bypass any/all AV. Code <a href="https://github.com/br-sn/CheekyBlinder" target="_blank">here</a>.</li>
<li><a href="https://github.com/sujaysathya/bunk_bot" target="_blank">bunk_bot</a> is built on Python and its main purpose is to attend your online classes for you. I've included it here despite it not being directly relevant to security because the techniques it uses to accomplish its goals can be applied to security problems. Think automating user creation or automating AV/EDR testing.</li>
<li><a href="https://www.tomanthony.co.uk/blog/zoom-security-exploit-crack-private-meeting-passwords/" target="_blank">Zoom Security Exploit – Cracking private meeting passwords</a>. 6 digit passwords and no rate limiting. What could possibly go wrong? The technique of simply removing the CSRF header surprisingly worked, and goes to show that you can never assume that because a control appears to be in place it is actually being enforced without checking.</li>
<li><a href="https://ijustwannared.team/2020/08/01/the-curious-case-of-aspnet_compiler-exe/" target="_blank">The Curious Case of Aspnet_Compiler.exe</a>. This post explores code execution with the Microsoft signed executable aspnet_compiler.exe. Another <a href="https://lolbas-project.github.io/#" target="_blank">LOLBin</a>!</li>
<li><a href="https://www.n00py.io/2020/08/password-spraying-secure-logon-for-f5-networks/" target="_blank">Password Spraying Secure Logon for F5 Networks</a> shows how to use a macro in burp suite to handle custom session rules. This method of request modification is generally applicable to lots of scenarios when brute forcing web logins.</li>
<li><a href="https://www.praetorian.com/blog/active-directory-computer-account-smb-relaying-attack" target="_blank">Active Directory Computer Account SMB Relaying Attack</a>. "SMB relaying is old hat" I hear you saying, which is true, but this specific flavor is new. With only unprivileged domain creds, an attacker can use the print spooler on one victim machine to authenticate to an attacker controlled relay which in turn allows attacker access as system to the second victim - assuming the first victim has admin rights to the second. The workhorse of this attack is <a href="https://gist.github.com/3xocyte/cfaf8a34f76569a8251bde65fe69dccc" target="_blank">this script</a>.</li>
<li><a href="http://blog.redxorblue.com/2020/07/one-click-to-compromise-fun-with.html" target="_blank">One Click to Compromise -- Fun With ClickOnce Deployment Manifests</a>. This was presented at BlackHat last year but just like the author, I missed it as well. One click + a security dialog = RCE. Test this out and it might become a new phishing technique to add to your toolkit.</li>
</ul>
</section>
<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://a2nkf.github.io/unauthd_Logic_bugs_FTW/" target="_blank">Unauthd - Logic bugs FTW</a>. Security researcher Ilias Morad aka <a href="https://twitter.com/a2nkf_" target="_blank">@A2nkF_</a>, describes a lovely exploit chain, composed of several security vulnerabilities he uncovered in macOS. This chain creates a local privilege escalation (LPE), from user all the way to the kernel! Code <a href="https://github.com/A2nkF/unauthd" target="_blank">here</a>. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1068/" target="_blank">T1068 Exploitation for Privilege Escalation</a>]</li>
<li><a href="https://github.com/Flangvik/SharpAppLocker" target="_blank">SharpAppLocker</a> is a C# port of the Get-AppLockerPolicy PS cmdlet. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1082/" target="_blank">T1082 System Information Discovery</a>]</li>
<li><a href="https://github.com/hvmi/hvmi" target="_blank">hvmi</a> - Hypervisor Memory Introspection. The main purpose of this project is to provide unmatched security from outside the virtual machine, by leveraging the hardware isolation provided by Intel VT-x. This is a super powerful capability, and a good post explaining how it works and what can be done with is is available <a href="https://hvmi.github.io/blog/introspection/2020/07/30/introduction.html" target="_blank">here</a>.</li>
<li><a href="https://www.dell.com/support/article/en-us/sln322125/dsa-2020-128-idrac-local-file-inclusion-vulnerability?lang=en" target="_blank">DSA-2020-128: iDRAC Local File Inclusion Vulnerability</a>. If your iDRAC isn't on a locked down subnet with monitored access, you're doing it wrong. This vulnerability will likely bear fruit on internal assessments for years to come. No PoC yet.</li>
<li><a href="https://github.com/crhenr/freki" target="_blank">freki</a> is a self-hosted malware analysis tool which is able to quickly statically analyze malware, integrate with VirusTotal, and perform pattern matching with Yara rules.</li>
<li><a href="https://github.com/anthemtotheego/C_Shot" target="_blank">C_Shot</a> is an offensive security tool written in C which is designed to download, inject, and execute shellcode in memory - either its own process or a child process using parent process spoofing. The <a href="http://blog.redxorblue.com/2020/07/cshot-just-what-doctor-ordered.html" target="_blank">blog post</a> shows how it can evade advanced EDR like CrowdStrike Falcon. I'm a bit surprised that it wasn't detected as it uses standard API calls to do APC process injection (VirtualAlloc, WriteProcessMemory, QueueUserAPC, ResumeThread). Pair this with direct syscalls for a likely more effective bypass. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1055/004/" target="_blank">T1055.004 Process Injection: Asynchronous Procedure Call</a>]</li>
<li><a href="https://github.com/nccgroup/winstrument" target="_blank">Winstrument</a> is a framework of modular scripts to aid in instrumenting Windows software using Frida for reverse engineering and attack surface analysis. Winstrument makes analyzing Windows apps easy, helping you quickly identify application functionality that might be insecure or warrant further review and could be useful on your next app assessment, or for finding a vulnerability in a Windows application on your next network assessment. Blog post <a href="https://research.nccgroup.com/2020/07/29/tool-release-winstrument-an-instrumentation-framework-for-windows-application-assessments/" target="_blank">here</a>.</li>
<li><a href="https://github.com/irsl/CVE-2020-1313" target="_blank">CVE-2020-1313</a>. You thought there would be a LWiS without a Windows LPE? This is a proof of concept for an "older" (patched June 2020) vulnerability in the Windows Update Orchestrator Service. It won't give you an instant win, but will execute at 2320 or within 3 days.</li>
</ul>

<section id="new-to-me">
<h2>New to Me</h2>
<p>This section is for news, techniques, and tools that weren't released last week but are new to me. Perhaps you missed them too!</p>
<ul>
<li><a href="https://github.com/cr0hn/festin" target="_blank">festin</a> is a tool for discovering open S3 Buckets starting from a domain.</li>
<li><a href="https://github.com/IncideDigital/Mistica" target="_blank">Mística</a> is a tool that allows to embed data into application layer protocol fields, with the goal of establishing a bi-directional channel for arbitrary communications. Currently, encapsulation into HTTP, DNS and ICMP protocols has been implemented, but more protocols are expected to be introduced in the near future. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1048/003/" target="_blank">T1048.003 Exfiltration Over Alternative Protocol: Exfiltration Over Unencrypted/Obfuscated Non-C2 Protocol</a>]</li>
<li><a href="https://github.com/ElevenPaths/ATTPwn" target="_blank">ATTPwn</a> is a computer security tool designed to emulate adversaries. The tool aims to bring emulation of a real threat into closer contact with implementations based on the techniques and tactics from the MITRE ATT&amp;CK framework.</li>
<li><a href="https://github.com/ventoy/Ventoy" target="_blank">Ventoy</a> is an open source tool to create bootable USB drive for ISO files. Stop formatting the same USB over and over for the OS you need at the time, and just store and boot from a list of ISOs.</li>
<li><a href="https://github.com/safing/portmaster/" target="_blank">portmaster</a> is a privacy app that at its core simply intercepts all your network connections. Think of a pi-hole for your computer. Or an ad-blocker that blocks ads on your whole computer, not only on your browser. This is an open-source version of <a href="https://www.obdev.at/products/littlesnitch/index.html" target="_blank">Little Snitch</a> or <a href="https://www.glasswire.com/" target="_blank">GlassWire</a>. Still in alpha, but something to keep an eye on.</li>
<li><a href="https://www.elttam.com/blog/env/" target="_blank">Hacking with environment variables</a> is a quick post showing some little known environment variables that can be used to execute code in unexpected places.</li>
<li><a href="https://github.com/Ciphey/Ciphey" target="_blank">Ciphey</a> is a fully automated decryption tool using natural language processing &amp; artificial intelligence, along with some common sense.</li>
</ul>
