<p>Last Week in Security is a summary of the interesting cybersecurity news, techniques, tools and exploits from the previous week. This post covers 2020-07-06 to 2020-07-13.
MITRE ATT&amp;CK techniques are in brackets where appropriate.</p>
<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://www.zdnet.com/article/mozilla-suspends-firefox-send-service-while-it-addresses-malware-abuse/" target="_blank">Mozilla suspends Firefox Send service while it addresses malware abuse</a>. The popular end-to-end encrypted file transfer service is "temporarily" offline as Mozilla works to implement a better abuse reporting system. However, the code for Send is <a href="https://github.com/mozilla/send" target="_blank">open source</a> so you can run your own instance if you rely on the service. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1204/001/" target="_blank">T1204.001 User Execution: Malicious Link</a>]</li>
<li><a href="https://www.digitalshadows.com/blog-and-research/from-exposure-to-takeover-part-1-beg-borrow-and-steal-your-way-in/" target="_blank">From Exposure To Takeover: Part 1. Beg, Borrow, And Steal Your Way In</a>. The number of stolen credentials has gone up 300% over the last two years. If you haven't enabled multi-factor authentication everywhere possible (bonus points for non-SMS) you should! [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1078/" target="_blank">T1078 Valid Accounts</a>]</li>
<li><a href="https://www.microsoft.com/en-us/research/blog/toward-trusted-sensing-for-the-cloud-introducing-project-freta/" target="_blank">Toward trusted sensing for the cloud: Introducing Project Freta</a> introduces a system to provide automated full-system volatile memory inspection of Linux systems. More information available in <a href="https://docs.microsoft.com/en-us/security/research/project-freta/" target="_blank">the documentation</a>.</li>
<li><a href="https://research.nccgroup.com/2020/07/12/understanding-the-root-cause-of-f5-networks-k52145254-tmui-rce-vulnerability-cve-2020-5902/" target="_blank">Understanding the root cause of F5 Networks K52145254: TMUI RCE vulnerability CVE-2020-5902</a>. Last week's F5/BIG-IP vulnerability and exploit got a lot of attention. This posts breaks down the root cause and may help you find similar issues, or just have a better understanding of what happened when someone asks.</li>
</ul>
</section>
<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://www.certego.net/en/news/advanced-vba-macros/" target="_blank">Advanced VBA macros: bypassing olevba static analyses with 0 hits</a>. In this post <a href="https://twitter.com/gabriele_pippi" target="_blank">@gabriele_pippi</a> shows multiple techniques to bypass static detection of VBA macros in Word and Excel documents. Reference this on your next phishing assessment to help get around pesky mail or file filters. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1562/" target="_blank">T1562 Impair Defenses</a>]</li>
<li><a href="https://modexp.wordpress.com/2020/07/07/wpi-wm-paste/" target="_blank">Windows Process Injection: EM_GETHANDLE, WM_PASTE and EM_SETWORDBREAKPROC</a>. This post digs up nearly 20 year old research to come up with a novel process injection method for Windows that uses the clipboard to copy shellcode to the target process. Odzhan adds <a href="https://github.com/odzhan/injection/blob/master/eminject/poc.c" target="_blank">this new technique</a> to their <a href="https://github.com/odzhan/injection" target="_blank">expansive library</a> of injection techniques. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1055/" target="_blank">T1055 Process Injection</a>]</li>
<li><a href="https://medium.com/palantir/restricting-smb-based-lateral-movement-in-a-windows-environment-ed033b888721" target="_blank">Restricting SMB-based lateral movement in a Windows environment</a>. This is pure gold for blue teams or red teams writing remediation recommendations for SMB attacks. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1021/002/" target="_blank">T1021.002 Remote Services: SMB/Windows Admin Shares</a>]</li>
<li><a href="https://www.praetorian.com/blog/praetorians-approach-to-red-team-infrastructure" target="_blank">Praetorian's Approach to Red Team Infrastructure</a>. Praetorian takes a modern, "BeyondCorp" or "Zero Trust" approach to their red team infrastructure. I particularly like the use of hardware tokens for authentication in this reference architecture. If you are a red team, have you red-teamed your own infrastructure?</li>
<li><a href="https://medium.com/bugbountywriteup/how-i-was-able-to-change-victims-password-using-idn-homograph-attack-587111843aff" target="_blank">How I was able to change victim’s password using IDN Homograph Attack</a>. This is a "simple" yet effective technique that exploits a system's poor processing of internationalized domain name (IDN) homographs. The use of burp collaborator to exploit the issue is well done, and prevents the need to buy a domain.</li>
<li><a href="https://redteaming.co.uk/2020/07/12/dll-proxy-loading-your-favorite-c-implant/" target="_blank">DLL Proxy Loading Your Favorite C# Implant</a>. <a href="https://twitter.com/Flangvik" target="_blank">@Flangvik</a> shows the full process of abusing DLL proxying to deploy persistence of arbitrary C# (or really any position independent executable/shellcode) using <a href="https://github.com/Flangvik/SharpDllProxy" target="_blank">SharpDllProxy</a> and <a href="https://github.com/TheWover/donut" target="_blank">donut</a>. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1574/002/" target="_blank">T1574.002 Hijack Execution Flow: DLL Side-Loading</a>]</li>
<li><a href="http://jackson-t.ca/edr-reversing-evading-01.html" target="_blank">A Guide to Reversing and Evading EDRs: Parts 1-3</a>. <a href="https://twitter.com/Jackson_T" target="_blank">@Jackson_T</a> delivers some seriously high quality research into the initial stages of setting up an EDR lab for research into evasion techniques, including diverting and inspecting the telemetry connections. If you have any interest in EDR evasion (I most certainly do), follow this series closely. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1562/" target="_blank">T1562 Impair Defenses</a>]</li>
<li><a href="https://m417z.com/Introducing-Winbindex-the-Windows-Binaries-Index/" target="_blank">Introducing Winbindex - the Windows Binaries Index</a>. <a href="https://twitter.com/m417z" target="_blank">@m417z</a> indexed all Windows files which appear in Windows update packages, and created a <a href="https://m417z.com/winbindex/" target="_blank">website</a> which allows to quickly view information about the files and download some of them from Microsoft servers. This will prove very useful to patch differs everywhere!</li>
</ul>
</section>
<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://www.zerodayinitiative.com/blog/2020/7/8/cve-2020-1300-remote-code-execution-through-microsoft-windows-cab-files" target="_blank">CVE-2020-1300: Remote Code Execution Through Microsoft Windows CAB Files</a>. The PathCchCanonicalize function <a href="https://research.checkpoint.com/2020/reverse-rdp-the-path-not-taken/" target="_blank">strikes again</a>, this time when opening a CAB file or installing a printer. In the case of the printer, arbitrary file write (and therefore remote code execution) is achieved as SYSTEM. No public proof of concepts exist yet, but there is enough detail in the post to craft your own.</li>
<li><a href="https://fortynorthsecurity.com/blog/sql-client-post/" target="_blank">Incoming .NET SQLClient</a> introduces a .NET MSSQL client that can be used with any in-memory .NET loader (i.e. Cobalt Strike's execute-assembly) for use during post-exploitation activities. [<a class="m-text m-dim" href="https://attack.mitre.org/tactics/TA0009/" target="_blank">TA0009 Collection</a>]</li>
<li><a href="https://github.com/google/tsunami-security-scanner" target="_blank">tsunami-security-scanner</a> is a general purpose network security scanner with an extensible plugin system for detecting high severity vulnerabilities with high confidence from Google. This scanner and <a href="https://github.com/projectdiscovery/nuclei" target="_blank">nuclei</a> are recent open source competitors to Tenable's Nessus.</li>
<li><a href="https://github.com/wintrmvte/SNOWCRASH" target="_blank">SNOWCRASH</a> creates a script that can be launched on both Linux and Windows machines. Payload selected by the user (in this case combined Bash and Powershell code) is embedded into a single polyglot template, which is platform-agnostic. I could see this being used for a cross-platform stage 1 payload against cross-platform target applications (i.e. Apache struts). [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1059/" target="_blank">T1059 Command and Scripting Interpreter</a>]</li>
<li><a href="https://github.com/glinares/VBA-Stendhal" target="_blank">VBA-Stendhal</a> injects encrypted commands into EMF shapes for C2 in VBA/Office malware. This is really cool Macro/VBA work by <a href="https://twitter.com/Laughing_Mantis" target="_blank">@Laughing_Mantis</a>, and you can read more about how the technique works <a href="https://medium.com/@laughing_mantis/malicious-shapes-in-office-part-2-910375cd05f3" target="_blank">here</a>. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1562/" target="_blank">T1562 Impair Defenses</a>]</li>
<li><a href="https://github.com/dalvarezperez/CreateFile_based_rootkit" target="_blank">CreateFile_based_rootkit</a> - Windows' DOS origins rears its ugly head once again with a bug where NtCreateFile can access directories with strange names like " ." but CreateFile cannot - it returns " " instead. This could prove useful if EDR tools use CreateFile, and also hides directories from explorer.exe and cmd.exe. [<a class="m-text m-dim" href="https://attack.mitre.org/techniques/T1562/" target="_blank">T1562 Impair Defenses</a>]</li>
</ul>
</section>
<section id="new-to-me">
<h2>New to Me</h2>
<p>This section is for news, techniques, and tools that weren't released last week but are new to me. Perhaps you missed them too!</p>
<ul>
<li><a href="https://github.com/intelowlproject/IntelOwl/" target="_blank">IntelOwl</a> - analyze files, domains, IPs in multiple ways from a single API at scale.</li>
<li><a href="https://www.freenom.com/" target="_blank">freenom.com</a> is a free domain registrar for .tk, .ml, .ga, .cf, and .gq domains. This could be useful for bug bounties or other engagements.</li>
<li><a href="https://github.com/hugsy/CFB" target="_blank">CFB</a> is a distributed tool for capturing I/O request packets (IRPs) sent to any Windows driver. This is very useful for driver reverse engineering and fuzzing. Check out the examples <a href="https://github.com/hugsy/CFB-cli" target="_blank">here</a>.</li>
</ul>
