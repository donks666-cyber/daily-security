<p>Last Week in Security is a summary of the interesting cybersecurity news, techniques, tools and exploits from the previous week. This post covers 2021-09-28 to 2021-10-06.</p>
<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://blog.cloudflare.com/october-2021-facebook-outage/" target="_blank">Understanding How Facebook Disappeared from the Internet</a>. Facebook had five hours of outage which is $9,806,639 of revenue based on 2020 numbers. A BGP issue took DNS down, and without DNS nothing worked. Bonus points if you have abandoned facebook and facebook owned services so much you didn't really notice. Facebook has issued their own <a href="https://engineering.fb.com/2021/10/05/networking-traffic/outage-details/" target="_blank">details about the October 4 outage</a>.</li>
<li><a href="https://oag.ca.gov/system/files/09-24-2021%20Customer%20Notification.pdf" target="_blank">Unauthorized Access to Your Coinbase Account</a>. Add this to the list of why SMS two factor is not real two factor, although it isn't clear if Coinbase or mobile carriers were the ultimate culprit. Either way, hardware security tokens are the answer.</li>
<li><a href="https://www.vice.com/en/article/z3xpm8/company-that-routes-billions-of-text-messages-quietly-says-it-was-hacked" target="_blank">Company That Routes Billions of Text Messages Quietly Says It Was Hacked</a>. Don your tin foil hat and conjecture if this is related to the previous story - probably not, this feels like good old fashion espionage.</li>
<li><a href="https://security.googleblog.com/2021/10/introducing-secure-open-source-pilot.html" target="_blank">Introducing the Secure Open Source Pilot Program</a>. Bug bounty for preventative security improvements is a great idea, but the ambiguity of the categories is even worse than normal bug bounty. Good luck to whomever has to "triage" these reports, but kudos to Google for trying something.</li>
<li><a href="https://www.vice.com/en/article/dyvnjk/twitch-hack-of-135-gb-of-data-includes-how-much-its-biggest-streamers-make" target="_blank">Twitch Hack of 135 GB of Data Includes How Much Its Biggest Streamers Make</a>. There are 6,000+ git repos as well. An interesting look behind the curtain of a major tech company.</li>
<li><a href="https://www.unicorn-engine.org/Unicorn2-beta" target="_blank">Unicorn2</a>. Unicorn, the popular CPU emulation framework is over 6 years old now and the second major version has been build from scratch on top of Qemu 5. It also adds PowerPC and RISCV emulation.</li>
</ul>
</section>
<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://posts.specterops.io/life-is-pane-persistence-via-preview-handlers-3c0216c5ef9e" target="_blank">Life is Pane: Persistence via Preview Handlers</a>. Windows explorer previews are generated by DLLs that are registered for each file extension. Attackers can register their own handlers or take over existing extensions for persistence when a user opens an explorer window containing a file with that extension. This is sneaky, and probably not detected by many EDR vendors... yet.</li>
<li><a href="https://swarm.ptsecurity.com/cisco-hyperflex-how-we-got-rce-through-login-form-and-other-findings/" target="_blank">Cisco Hyperflex: How We Got RCE Through Login Form and Other Findings</a>. The auth endpoint calculated the password hash of the plaintext supplied by the user by calling a one line python script from Go and passing the plaintext as an argument. Why not calculate the hash in the Go binary? Less RCE that way I suppose...</li>
<li><a href="https://voidsec.com/crucial-mod-utility-lpe-cve-2021-41285/" target="_blank">Crucial’s MOD Utility LPE – CVE-2021-41285</a>. More "gaming" drivers, more LPEs. If you need system, target gaming drivers!</li>
<li><a href="https://jellevergeer.com/reverse-engineering-and-decrypting-cyberark-vault-credential-files/" target="_blank">Reverse engineering and decrypting CyberArk vault credential files</a>. This post is a technical deep-dive into CyberArk credential files and how the credentials stored in these files are encrypted and decrypted. The author discovered it was possible to reverse engineer the encryption and key generation algorithms and decrypt the encrypted vault password</li>
<li><a href="https://github.com/daem0nc0re/Abusing_Weak_ACL_on_Certificate_Templates" target="_blank">Abusing Weak ACL on Certificate Templates</a>. This is a walkthrough of the ESC4 attack described in <a href="https://posts.specterops.io/certified-pre-owned-d95910965cd2" target="_blank">Certified Pre-Owned</a>.</li>
<li><a href="https://gist.github.com/jackullrich/ce3b5263e05c88548b175f4f21909384" target="_blank">Single Step Encryption/Decryption</a>. Decrypt and run shellcode one instruction (or 16 byte block) at a time. This should help against memory forensics, but may be unstable with complex shellcode.</li>
</ul>
</section>
<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/trickster0/OffensiveRust" target="_blank">OffensiveRust</a> is a series of experiments in weaponizing Rust for implant development and general offensive operations.</li>
<li><a href="https://httpd.apache.org/security/vulnerabilities_24.html" target="_blank">Apache HTTP Server 2.4 vulnerabilities</a>. This is a path traversal vulnerability that can lead to RCE. PoC: <cite>curl --data "A=|id&gt;&gt;/tmp/x;uname$IFS-a&gt;&gt;/tmp/x" 'http://[IP]:[PORT]/cgi-bin/.%2e/.%2e/.%2e/.%2e/bin/sh' -vv</cite> (credit to <a href="https://twitter.com/hackerfantastic/status/1445531829985968137" target="_blank">@hackerfantastic</a>). Note that this only affects 2.4.49 (released 2021-09-15) due to <a href="https://github.com/apache/httpd/commit/6a5d3e006b8dc8aca1a267a8607864e1c3607f61#diff-6418f40952d9b5f8e2aa0b8789022a1d6b484c2f2300ded31547129f74295f1c" target="_blank">this commit</a> from August 2021. Test it out in the <a href="https://github.com/blasty/CVE-2021-41773" target="_blank">CVE-2021-41773 Playground</a>.</li>
<li><a href="https://gitlab.com/theepicpowner/dcom_av_exec" target="_blank">DCOM_AV_EXEC</a> allows for "diskless" lateral movement to a target on the same network via DCOM. The AV_Bypass_Framework_V3 creates a .NET shellcode runner (output as DLL) which can be used with the DCOM_AV_EXEC tool to bypass antivirus solutions like Microsoft Defender as all shellcode is AES encrypted and executed in memory.</li>
<li><a href="https://github.com/ARPSyndicate/kenzer" target="_blank">kenzer</a> performs automated web assets enumeration &amp; scanning.</li>
<li><a href="https://github.com/mm0r1/exploits/tree/master/php-filter-bypass" target="_blank">PHP 7.0-8.0 disable_functions bypass [user_filter]</a> is a 10 year old bug to get around <cite>disabled_functions</cite> set in php.ini and execute shell commands on the target webserver.</li>
<li><a href="https://github.com/login-securite/DonPAPI" target="_blank">DonPAPI</a>  dumps DPAPI credentials remotely.</li>
<li><a href="https://github.com/treebuilder/aad-sso-enum-brute-spray" target="_blank">aad-sso-enum-brute-spray</a> A PoC for the vulnerability that would, in theory, allow one to perform brute force or password spraying attacks against one or more AAD accounts without causing account lockout or generating log data, thereby making the attack invisible.</li>
</ul>
<p>-<a href="https://github.com/p0dalirius/FindUncommonshares" target="_blank">FindUncommonShares</a> is a Python equivalent of PowerView's Invoke-ShareFinder.ps1 which finds uncommon SMB shares on remote machines.</p>
