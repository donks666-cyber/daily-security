<p>Last Week in Security is a summary of the interesting cybersecurity news, techniques, tools and exploits from the previous week. This post covers 2020-09-21 to 2020-09-28.</p>
<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://www.zdnet.com/article/russia-wants-to-ban-the-use-of-secure-protocols-such-as-tls-1-3-doh-dot-esni/" target="_blank">Russia wants to ban the use of secure protocols such as TLS 1.3, DoH, DoT, ESNI</a>. First we saw <a href="https://www.zdnet.com/article/china-is-now-blocking-all-encrypted-https-traffic-using-tls-1-3-and-esni/" target="_blank">China</a> block TLS 1.3 with ESNI, and now Russia is looking to do the same. Perhaps the Kremlin watched <a href="https://youtu.be/TDg092qe50g" target="_blank">my DEF CON talk</a> about how ESNI can be used for covert C2?</li>
<li><a href="https://research.checkpoint.com/2020/instagram_rce-code-execution-vulnerability-in-instagram-app-for-android-and-ios/" target="_blank">#Instagram_RCE: Code Execution Vulnerability in Instagram App for Android and iOS</a>. This vulnerability calls back to the 2015 classic <a href="https://googleprojectzero.blogspot.com/2015/09/stagefrightened.html" target="_blank">Stagefright</a> as it exploits an overflow in a file parser. While the researchers never got to full RCE, they did impressive Google Project Zero level work.</li>
<li><a href="https://www.group-ib.com/blog/oldgremlin" target="_blank">Big Game Hunting: Now in Russia</a>. There has been a long standing unspoken rule in the ransomware underground - Don't mess with Russian companies. One crew finally got greedy enough to break that rule, and OldGremlin is deploying TinyCryptor against Russian targets.</li>
<li><a href="https://www.zdnet.com/article/mozilla-shuts-down-firefox-send-and-firefox-notes-services/" target="_blank">Mozilla shuts down Firefox Send and Firefox Notes services</a>. The struggling Mozilla permanently shuts down its popular "Send" service which allowed users to send large files with end-to-end encryption. This was quickly used to distribute malware. This was followed by Microsoft removing the ability for <a href="https://www.bleepingcomputer.com/news/microsoft/microsoft-removes-windows-defender-ability-after-security-concerns/" target="_blank">Windows Defender to download files</a> using the command line. Attackers will always target "legitimate" methods of file transfer to distribute malware.</li>
<li><a href="https://checkra.in/news/2020/09/iOS-14-announcement" target="_blank">The state of checkra1n on iOS 14</a>. The bootROM exploit for iOS devices is alive and well, with an initial release working on lots of devices and the latest iOS. This is the jailbreak that keeps on giving.</li>
<li><a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar20-268a" target="_blank">Federal Agency Compromised by Malicious Cyber Actor</a>. Despite the boring title, this report actually contains good information. Specific TTPs and IOCs are discussed and available.</li>
</ul>
</section>
<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://www.pentestpartners.com/security-blog/the-return-of-raining-system-shells-with-citrix-workspace-app/" target="_blank">The Return of Raining SYSTEM Shells with Citrix Workspace app</a>. The MSI transform feature is not widely known, but probably can be used malicious as shown here in many more cases. This is another great example of not giving up on a bug just because it was "patched." CCob also released <a href="https://github.com/CCob/PwnyForm" target="_blank">PwnyForm</a>, a tool that can take an MSI input and generate an MSI transform that can be used for arbitrary command execution via custom actions.</li>
<li><a href="https://medium.com/@_graphx/unquoted-paths-theyre-not-just-for-services-anymore-852681e740c9" target="_blank">Unquoted paths. They’re not just for services anymore</a>. Unqouted service paths are an all too common issues on Windows machines, especially with third party software. This post discusses how the same issue can affect Scheduled Tasks, and how a standard Microsoft task was vulnerable. Add this to your list of things to check for potential privilege escalation on a Windows machine.</li>
<li><a href="https://www.synacktiv.com/sites/default/files/2020-09/fortinet_sslvpn_client_lpe.pdf" target="_blank">Local Privilege Escalation in FortinetSSL VPN client for Linux</a>. Local Privilege Escalation (LPE) comes for Linux via the FortinetSSL VPN client which is SUID and blindly trusts argv[0] to be the path to its install location. This advisory contains a PoC which affects (at least) FortinetSSL Linux VPN client versions 4.0-2281 and 4.4-2336.</li>
<li><a href="https://blog.zsec.uk/zerologon-attacking-defending/" target="_blank">ZeroLogon(CVE-2020-1472) - Attacking &amp; Defending</a>. As the dust settles from the ZeroLogon announcement, this post goes into the technical details of both the exploit (but no work on repairing the DC after exploit) and various detection strategies.</li>
<li><a href="https://dirkjanm.io/a-different-way-of-abusing-zerologon/" target="_blank">A different way of abusing Zerologon (CVE-2020-1472)</a>. Dirk-jan is one of the researchers on the forefront of Windows/AD knowledge (along with Will Schroeder [<a href="https://twitter.com/harmj0y" target="_blank">@harmj0y</a>] and Benjamin Delpy [<a href="https://twitter.com/gentilkiwi" target="_blank">@gentilkiwi</a>]) and this post proves it. Dirk-jan manages to modify impacket to use Zerologon to relay a machine authentication solicited with <a href="https://github.com/leechristensen/SpoolSample" target="_blank">SpoolSample</a> from one DC to another, preventing the machine account password reset in the normal Zerologon attack flow that causes major issues.</li>
<li><a href="https://www.samba.org/samba/security/CVE-2020-1472.html" target="_blank">Samba Unauthenticated domain takeover via netlogon ("ZeroLogon")</a>. If you are running Samba &lt;= 4.7 as a domain controller, you are vulnerable to ZeroLogon. This bug isn't just for Windows!</li>
<li><a href="https://decoder.cloud/2020/09/23/abusing-group-policy-caching/" target="_blank">Abusing Group Policy Caching</a>. A standard domain user can perform, via the “gpsvc” service, arbitrary file overwrite with SYSTEM privileges by altering behavior of “Group Policy Caching”. This can be used to elevate from a standard user to SYSTEM, however it has since been patched (<a href="https://portal.msrc.microsoft.com/en-us/security-guidance/advisory/CVE-2020-1317" target="_blank">CVE-2020-1317</a>) by making the group policy folders readonly for standard users.</li>
<li><a href="https://medium.com/@curtbraz/phishing-your-password-manager-8e8f90e52918" target="_blank">Phishing Your Password Manager</a>. Feeling smug about credential harvesting because your password manager recognizes when sites don't match? Curtis Brazzell shows that some auth providers allow custom login pages which can be implanted with form capture so the domain does match, and your password manager likely will fill in creds. Manually verify those domains, every time!</li>
<li><a href="https://medium.com/@prizmant/hacking-punkbuster-e22e6cf2f36e" target="_blank">Hacking Punkbuster</a>. Punkbuster is an anti-cheat system for PC video games that has some very malware like properties (ability to take screenshots). This post shows how the Punkbuster server is vulnerable to a path traversal in the screenshot name. Unfortunately, "for security reasons we will only give a high level description of the vulnerability and won’t dive deep into the actual reverse engineering process." I'm guessing they used <a href="https://frida.re/" target="_blank">Firda</a>?</li>
<li><a href="https://payatu.com/blog/arun/bypassing-android-mdm-using-electromagnetic-fault-injection-by-a-gas-lighter-for-$1.5$">Bypassing Android MDM using Electromagnetic Fault Injection by a Gas Lighter for $1.5</a>. We've talked about hardware glitching before (<a href="https://blog.badsectorlabs.com/last-week-in-security-lwis-2020-08-31.html">LWiS 2020-08-31</a>), but this is a low cost, low complexity attack to glitch an Android phone into sysdump which allows for the System on a Chip (SoC) recovery port to be used to flash new firmware, bypassing the very locked down system. Physical access is <em>always</em> root access, sometimes it just takes a bit more work.</li>
<li><a href="https://www.trustedsec.com/blog/azure-account-hijacking-using-mimikatzs-lsadumpsetntlm/" target="_blank">Azure Account Hijacking using mimikatz’s lsadump::setntlm</a>. Have Domain Admin but need to access data as a user with a password hash that won't crack? Wait for them to go home, then use <cite>lsadump::changentlm</cite> and <cite>lsadump::setntlm</cite> to change their password to something you do know, and put it back to their NTLM hash before they return in the morning.</li>
<li><a href="https://www.digeex.de/blog/tinytinyrss/" target="_blank">Exploiting Tiny Tiny RSS</a>. This is a deep dive into webapp source code review and hacking. The authors pull together a very impressive single click exploit that backdoor's a user's TT-RSS server. There are some great advanced techniques in this post - and the classic <cite>gopher://</cite> trick as well.</li>
<li><a href="https://www.matteomalvica.com/blog/2020/09/24/weaponizing-cve-2020-17382/" target="_blank">Kernel exploitation: weaponizing CVE-2020-17382 MSI Ambient Link driver</a>. If you have any interest in kernel or driver exploitation, this blog is a must read.</li>
<li><a href="https://www.solomonsklash.io/smaller-c-payloads-on-windows.html" target="_blank">Smaller C Payloads on Window</a> shows how to get an 8-12x reduction in binary size by removing the Visual C++ runtime from Windows binaries, and only including the functionality your program actually needs. Note that this limits some of the comfort features of C/C++.</li>
</ul>

<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/corkami/mitra" target="_blank">mitra</a> is a python tool to generate binary polygots (one file that can be parsed as two different formats like a PNG and a DICOM). This could be useful to bypass file upload restrictions or generally mess with file parsers.</li>
<li><a href="https://github.com/EncodeGroup/Gopher" target="_blank">Gopher</a> is a C# tool to search for credentials from all types of applications on Windows.</li>
<li><a href="https://github.com/EncodeGroup/SharpDirLister" target="_blank">SharpDirLister</a> is a .NET 4.0 tool for super fast directory listings.</li>
<li><a href="https://github.com/EncodeGroup/RegSave" target="_blank">RegSave</a> is a .NET 3.5 tool for dumping the SAM, SYSTEM, and SECURITY registry keys.</li>
<li><a href="https://github.com/goichot/CVE-2020-3433" target="_blank">CVE-2020-3433</a> is a collection of 3 vulnerabilities in the Cisco AnyConnect client for Windows, one of which is a local privilege escalation (sound familiar? <a href="https://github.com/goichot/CVE-2020-3153" target="_blank">CVE-2020-3153</a> was exploited by the same researcher - <a href="https://twitter.com/AntoineGoichot" target="_blank">@AntoineGoichot</a> after being found by <a href="https://twitter.com/yorickkoster" target="_blank">@yorickkoster</a>).</li>
<li><a href="https://offensive-terraform.github.io/" target="_blank">Offensive Terraform Modules</a> provide automated multi-step offensive attack modules using Infrastructure as Code (IAC).</li>
<li><a href="https://github.com/muesli/duf" target="_blank">duf</a> is a clean disk usage utility for Linux, FreeBSD, and macOS. It even outputs in JSON!</li>
<li><a href="https://github.com/vulnersCom/mikrot8over" target="_blank">mikrot8over</a> a rework of the Mikrotik RouterOS (&lt;= 6.38.4) exploit with multithreaded scan capability.</li>
<li><a href="https://github.com/slyd0g/UrbanBishopLocal" target="_blank">UrbanBishopLocal</a> is a port of FuzzySecurity's <a href="https://github.com/FuzzySecurity/Sharp-Suite#urbanbishop" target="_blank">UrbanBishop</a> project for inline shellcode execution.</li>
<li><a href="https://www.eff.org/deeplinks/2020/09/introducing-yaya-new-threat-hunting-tool-eff-threat-lab" target="_blank">Introducing “YAYA”, a New Threat Hunting Tool From EFF Threat Lab</a>. YAYA is a new open source tool to help researchers manage multiple YARA rule repositories. YAYA starts by importing a set of high-quality YARA rules and then lets researchers add their own rules, disable specific rulesets, and run scans of files.</li>
</ul>
</section>
<section id="new-to-me">
<h2>New to Me</h2>
<p>This section is for news, techniques, and tools that weren't released last week but are new to me. Perhaps you missed them too!</p>
<ul>
<li><a href="https://labs.redyops.com/index.php/2020/09/11/ibm-qradar-wincollect-escalation-of-privileges-cve-2020-4485-cve-2020-4486/" target="_blank">IBM QRadar Wincollect Escalation of Privilege (CVE-2020-4485 &amp; CVE-2020-4486)</a>. I missed this one last week, but arbitrary file delete leads to elevation of privilege using the IBM QRadar Wincollect installer.</li>
<li><a href="https://github.com/crowdsecurity/crowdsec" target="_blank">crowdsec</a> is a lightweight Go agent to detect and respond to bad behaviors. It also automatically benefits from our global community-wide IP reputation database (can be disabled with <cite>apimode: false</cite>). If fail2ban is too simple, this may be what you are looking for.</li>
<li><a href="https://github.com/michaelweber/Macrome" target="_blank">Macrome</a> is an Excel macro document reader and writer for red teamers and analysts. It uses Excel 4.0 macros and the BIFF8 (Excel 97-2003 Binary) XLS format which will likely bypass many detections. Details <a href="https://malware.pizza/2020/05/12/evading-av-with-excel-macros-and-biff8-xls/" target="_blank">here</a>.</li>
</ul>
