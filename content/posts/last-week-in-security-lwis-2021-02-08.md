<section id="news">
<h2>News</h2>

<li><a href="https://www.trustwave.com/en-us/resources/blogs/spiderlabs-blog/full-system-control-with-new-solarwinds-orion-based-and-serv-u-ftp-vulnerabilities/" target="_blank">Full System Control with New SolarWinds Orion-based and Serv-U FTP Vulnerabilities</a>. This doesn't surprise me. As soon as the initial SolarWinds news broke, security researchers set their sights on it - with the inevitable outcome of more vulnerabilities. This pattern plays out with most high profile software vulnerabilities.</li>
<li><a href="https://www-nytimes-com.cdn.ampproject.org/c/s/www.nytimes.com/2021/02/06/technology/cyber-hackers-usa.amp.html" target="_blank">How the United States Lost to Hackers - America’s biggest vulnerability in cyberwarfare is hubris</a>. <section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://www.optiv.com/explore-optiv-insights/source-zero/endpoint-detection-and-response-how-hackers-have-evolved" target="_blank">Endpoint Detection and Response: How Hackers Have Evolved</a>. This blog is so good it has a <a href="https://www.optiv.com/explore-optiv-insights/source-zero/edr-and-blending-how-attackers-avoid-getting-caught" target="_blank">second part</a>. It goes over user-mode hooking and direct syscalls, but introduces some unique bypasses, side-by-side loading vs process injection, and the use of code signing and spoofed attributes via syso files. The <a href="https://github.com/optiv/ScareCrow" target="_blank">ScareCrow</a> tool that automates all this is well done and even can provide phishing-ready one liners or a macro payload.</li>
<li><a href="https://supercookie.me/workwise" target="_blank">supercookie</a> uses favicons to assign a unique identifier to website visitors. Unlike traditional tracking methods, this ID can be stored almost persistently and cannot be easily cleared by the user. The tracking method works even in the browser's incognito mode and is not cleared by flushing the cache, closing the browser or restarting the system, using a VPN or installing AdBlockers. Scary stuff. You set up your tracking instance with the <a href="https://github.com/jonasstrehle/supercookie" target="_blank">open sourced backend</a>.</li>
<li><a href="https://raxis.com/blog/rd-web-access-vulnerability" target="_blank">Microsoft Remote Desktop Web Access Authentication Timing Attack</a>. The combination of returning the Active Directory Domain in an RPC response and a timing attack disclosing valid usernames allows a remote attacker with no knowledge of the target to enumerate valid Active Directory usernames using only RDWeb.</li>
<li><a href="https://isc.sans.edu/forums/diary/Abusing+Google+Chrome+extension+syncing+for+data+exfiltration+and+CC/27066/" target="_blank">Abusing Google Chrome extension syncing for data exfiltration and C&amp;C</a>. The Google Chrome Sync feature allows for a decent amount of data to be exfiltrated from an infected browser. This plus the ability for extensions to snoop on all pages makes Chrome Extensions a natural attack vector (think oath tokens, cookies, etc). This has been happening for <a href="https://decoded.avast.io/janvojtesek/backdoored-browser-extensions-hid-malicious-traffic-in-analytics-requests/" target="_blank">years</a>.</li>
<li><a href="https://www.contextis.com/en/blog/dynamicwrapperex-windows-api-invocation-from-windows-script-host" target="_blank">DynamicWrapperEx – Windows API Invocation from Windows Script Host</a>. This blog post will covers some basics of COM, how to leverage OLE Automation, the x64 standard calling convention, registration-free activation, and some of the limitations and security considerations around the use of a new <a href="https://github.com/ctxis/DynamicWrapperEx" target="_blank">tool</a> published with the post.</li>
<li><a href="https://blog.thecybersecuritytutor.com/spoofing-and-attacking-with-skype/" target="_blank">Spoofing and Attacking With Skype</a>. Tampering with requests to Skype allows for the creation of some pretty serious social engineering payloads.</li>
<li><a href="https://www.trustedsec.com/blog/injecting-rogue-dns-records-using-dhcp/" target="_blank">Injecting Rogue DNS Records Using DHCP</a>. Some environments automatically create DNS entries for DHCP host names which are user controlled. How can this be used? From the post - The first step would be to launch Responder in analysis mode (no poisoning) and listen for hosts broadcasting queries for hosts that they can’t resolve. Next, pick the most common one, inject a DNS record with the method above, and wait for an inbound connection from the querying system. If that account has a weak password, you might be able to crack it or potentially forward it is using NetNTLMRelayX.</li>
<li><a href="https://dayzerosec.com/posts/getting-started/" target="_blank">Getting Started with Exploit Development</a>. This is a good collection of resources as well as introduction to the topic.</li>
<li><a href="https://dhiyaneshgeek.github.io/bug/bounty/2020/02/06/recon-with-me/" target="_blank">Recon with Me !!!</a>. Using some really nice <a href="https://en.wikipedia.org/wiki/Unix_philosophy" target="_blank">"unix philosophy"</a> tools, this post shows how to construct your on continuous monitoring solution for bug bounties or for blue teams to use against themselves.</li>
<li><a href="https://luemmelsec.github.io/Relaying-101/" target="_blank">Relaying 101</a>. Need a quick overview of the <em>many</em> ways relaying can be used in a Windows environment? <a href="https://twitter.com/theluemmel" target="_blank">@theluemmel</a> has you covered, with a healthy dose of self-deprecating memes for good measure.</li>
<li><a href="https://igor-blue.github.io/2021/02/04/secure-boot.html" target="_blank">In-depth dive into the security features of the Intel/Windows platform secure boot process</a>. If you've ever wondered what really happens when you boot an Intel based Windows computer with SecureBoot enabled, this post will show you in great detail.</li>
<li><a href="https://cert.grnet.gr/en/blog/reverse-engineering-emotet/" target="_blank">Reverse engineering Emotet – Our approach to protect GRNET against the trojan</a>. This post is chock full of meaty reverse engineering detail. If you were ever curious how major botnets went from email opens to trojan'd machines without tripping AV/EDR, this post is for you.</li>
</ul>
</section>
<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/0xdea/raptor_infiltrate20" target="_blank">raptor_infiltrate20</a> contains multiple exploits (some 0days?) for Solaris 10 and 11. The <a href="https://vimeo.com/474793702" target="_blank">talk</a> is worth checking out as well. Even modern operating systems have some dumb logic bugs left.</li>
<li><a href="https://github.com/Patrowl/PatrowlHears" target="_blank">PatrowlHears</a> is an advanced and real-time Vulnerability Intelligence platform, including CVE, exploits and threats news. This commercially developed continuous monitoring system is now open source (AGPL)!</li>
<li><a href="https://github.com/forrest-orr/ExploitDev/blob/master/Exploits/Re-creations/Forrest_Orr_CVE-2020-0674_32bit.html" target="_blank">Forrest_Orr_CVE-2020-0674_32bit.html</a>. This is a Microsoft Internet Explorer 8-11 32-bit Use-After-Free exploit. It targets 32-bit vs the metasploit 64-bit only exploit which is important as IE is 32-bit by default on Windows 7 and 8.1. As with everything Forrest Orr does, it's a work of art and bypasses DEP, ASLR, and EMET 5.5 (EAF, EAF+, stack pivot protection, SimExec, and CallerCheck).</li>
<li><a href="https://github.com/owenrumney/squealer" target="_blank">squealer</a> scans a local git repository for secrets that are being leaked deep within the commit history using regular expressions. Use it on offensive engagements or as part of your CI to keep secrets out of git repositories.</li>
<li><a href="https://github.com/shantanu561993/Awesome_Firebase_DomainFront" target="_blank">Awesome_Firebase_DomainFront</a> is a repository for a Firebase domain fronting application. This is a demonstration application that uses firebase cloud functions to front traffic. You can use arbitrary *.web.app domains. See <a href="https://www.redteam.cafe/red-team/domain-front/firebase-domain-front-hiding-c2-as-app-traffic" target="_blank">the blog post</a> for more information.</li>
<li><a href="https://github.com/rvrsh3ll/SharpSMBSpray" target="_blank">SharpSMBSpray</a> sprays a hash via smb to check for local administrator access. With <a href="https://github.com/cube0x0/SharpMapExec" target="_blank">SharpMapExec</a> you'd have to specify a list of computer names vs a network range in SharpSMBSpray.</li>
<li><a href="https://github.com/silentsignal/rsa_sign2n" target="_blank">rsa_sign2n</a> contains experimental code to calculate RSA public keys based on two known message-signature pairs, code to extract and generate RSA and HMAC signatures for JWTs, and proof-of-concept code to exploit the CVE-2017-11424 key confusion vulnerability in pyJWT, without knowing the public key of the target (all details <a href="https://blog.silentsignal.eu/2021/02/08/abusing-jwt-public-keys-without-the-public-key/" target="_blank">here</a>).</li>
<li><a href="https://github.com/theevilbit/Shield/releases/tag/0.9" target="_blank">Shield</a> - not new, but with a shiny new EndpointSecurity entitlement from Apple it can be easily installed. More information on this injection attack protection tool <a href="https://theevilbit.github.io/shield/" target="_blank">here</a>.</li>
<li><a href="https://github.com/pkb1s/SharpRelay" target="_blank">SharpRelay</a> automates the process of loading the WinDivert driver (signed) and communicate with it to pull of an NTLM relay remotely through a Cobalt Strike beacon. If you don't want to load the driver, <a href="https://github.com/Kevin-Robertson/InveighZero" target="_blank">InveighZero</a> may do what you need.</li>
</ul>

<section id="new-to-me">
<h2>New to Me</h2>
<p>This section is for news, techniques, and tools that weren't released last week but are new to me. Perhaps you missed them too!</p>
<ul>
<li><a href="https://github.com/muraenateam/necrobrowser" target="_blank">necrobrowser</a> is a browser instrumentation microservice written in NodeJS: it uses the Puppeteer library to control instances of Chrome or Firefox in headless and GUI mode. The idea is to feed NecroBrowser with web sessions harvested during phishing campaigns (see Muraena) to quickly perform actions hijacking the victim session.</li>
<li><a href="https://github.com/evilsocket/uroboros" target="_blank">uroboros</a> is a GNU/Linux monitoring tool focused on single processes. This could be a useful first step for dynamic analysis of a Linux binary.</li>
<li><a href="https://rmusser.net/docs/Active_Directory.html" target="_blank">Information Security Reference</a> is a very complete resource for all things Information Security.</li>
</ul>
