<p>Last Week in Security is a summary of the interesting cybersecurity news, techniques, tools and exploits from the previous week. This post covers 2021-01-11 to 2021-01-18.</p>
<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://arstechnica.com/information-technology/2021/01/mimecast-says-hackers-stole-a-certificate-and-used-it-to-target-its-customers/" target="_blank">Hackers steal Mimecast certificate used to encrypt customers’ M365 traffic</a>. On the heals of the SolarWinds news, it feels like someone is going after "supply chain" type targets pretty hard.</li>
<li><a href="https://www.forbes.com/sites/rachelsandler/2021/01/15/so-many-people-are-using-signal-it-caused-an-outage/" target="_blank">So Many People Are Using Signal It Caused An Outage</a>. With WhatsApp's heavy handed, but in reality minor, privacy policy change users have been flocking to Signal. Perhaps 2021 will be the year of digital privacy? Other privacy minded services saw growth as <a href="https://www.zdnet.com/article/duckduckgo-surpasses-100-million-daily-search-queries-for-the-first-time/" target="_blank">DuckDuckGo surpasses 100 million daily search queries for the first time</a>.</li>
<li><a href="https://www.crowdstrike.com/blog/sunspot-malware-technical-analysis/" target="_blank">SUNSPOT: An Implant in the Build Process</a>. The SolarWinds fallout continues in this analysis of the implant that was used to backdoor the Orion software. Like much of the operation, this is top notch hacking. Sunspot waiting until Orion was being built to insert the backdoor, so code repositories never contained evidence. The number of mutexes and hash checks suggest a very thorough QA process for this. I wouldn't be surprise if somewhere there is a complete one-for-one Orion build environment in the basement of a government building.</li>
<li><a href="https://malvuln.com/" target="_blank">Malvuln - Finding and exploiting vulnerable Malware.</a>. This website and <a href="https://twitter.com/malvuln" target="_blank">@malvuln</a> twitter account are dedicated to finding vulnerabilities in malware. Detailed reports with PoCs are provided as well!</li>
<li><a href="https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2021-1647" target="_blank">Microsoft Defender Remote Code Execution Vulnerability</a>. A patch for this vulnerability (CVE-2021-1647) was part of January Patch Tuesday from Microsoft, and for normal users, Defender updates automatically in the background. For enterprises with Windows Server Update Services (WSUS), this should be a high priority patch. Any action that results in Defender scanning a malicious file could result in code execution as SYSTEM. While a <a href="https://www.youtube.com/watch?v=FlaYEnckGxU" target="_blank">PoC has been teased</a> [UPDATE: this is vulnerability researcher bait and totally fake!], no code is public yet.</li>
<li><a href="https://googleprojectzero.blogspot.com/2021/01/introducing-in-wild-series.html" target="_blank">Introducing the In-the-Wild Series</a>. Google's Project Zero team released six posts about in-the-wild zero days, breaking down the root causes and going into technical detail for each.</li>
<li><a href="https://www.bleepingcomputer.com/news/security/windows-10-bug-crashes-your-pc-when-you-access-this-location/" target="_blank">Windows 10 bug crashes your PC when you access this location</a>. Opening a file path in a browser (Chrome and Firefox have been reported to work) will blue screen a fully patched Windows 10 machine. 1995 called and want its bug back.</li>
</ul>
</section>
<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://github.com/httpvoid/writeups/blob/main/Apple-RCE.md" target="_blank">Finding 0day to hack Apple</a>. A CMS 0day lead to shells on a few Apple systems. This is a detailed walk through of how the vulnerabilities were found and chained, as well as the web application firewall (WAF) issues and bypasses.</li>
<li><a href="https://srcincite.io/blog/2021/01/12/making-clouds-rain-rce-in-office-365.html" target="_blank">Making Clouds Rain :: Remote Code Execution in Microsoft Office 365</a>. This exchange RCE bug doesn't want to die, as this is now the third patch bypass. As it stands it requires some administrative rights to an on-premise Exchange server (the post says DLP rights, but <a href="https://portswigger.net/daily-swig/critical-zero-day-rce-in-microsoft-office-365-awaits-third-security-patch" target="_blank">other sources</a> say that is not required).</li>
<li><a href="https://luemmelsec.github.io/Circumventing-Countermeasures-In-AD/" target="_blank">Sailing Past Security Measures In AD</a>. This post covers my favorite technique in red teaming: "glue." There are so many good tools and techniques out there, if you can glue a few together or modify them slightly, you can create some effective "custom" tooling. Part of the reason I write this blog is to bring these building blocks to light for myself and others to "glue" together.</li>
<li><a href="https://research.nccgroup.com/2021/01/15/sign-over-your-hashes-stealing-netntlm-hashes-via-outlook-signatures/" target="_blank">Sign over Your Hashes – Stealing NetNTLM Hashes via Outlook Signatures</a>. A 1x1 pixel image in the signature of a user's outlook client, which does not require elevated permissions or GUI access, can yield NTLM hashes from anyone who opens a message. <a href="https://github.com/nccgroup/nccfsas/tree/main/Tools/Sigwhatever" target="_blank">Sigwhatever</a> automates this whole process and cleanup too!</li>
<li><a href="https://secret.club/2021/01/15/bitlocker-bypass.html" target="_blank">BitLocker Lockscreen bypass</a>. This was patched on 2020-07-14, but is quite the trick none the less. By inserting a specially crafted USB stick and using a DLL hijack for the narrator, an attacker is able to get SYSTEM account code execution on a locked Windows 10 machine. This disclosure likely made many <a href="https://en.wikipedia.org/wiki/Evil_maid_attack" target="_blank">evil maids</a> very upset.</li>
<li><a href="https://secret.club/2021/01/14/vbox-escape.html" target="_blank">Escaping VirtualBox 6.1: Part 1</a>. If the 6 month old lockscreen bypass was too old news for you, secret club delivers a fresh 0day VirtualBox escape on Windows. A <a href="https://youtu.be/mjKxafMbpS0" target="_blank">demo</a> and <a href="https://github.com/Sauercloud/RWCTF21-VirtualBox-61-escape" target="_blank">PoC</a> are available.</li>
<li><a href="https://posts.specterops.io/local-privilege-escalation-in-vmware-vrealize-automation-vra-guest-agent-service-a83fbdce1129" target="_blank">Local Privilege Escalation in VMware vRealize Automation (vRA) Guest Agent Service</a>. This reads like a CTF challenge with the use of sloppy scripting and bad permissions. 3rd party services are the source of <em>many</em> local privilege escalation vulnerabilities on Windows.</li>
<li><a href="https://blog.cobaltstrike.com/2021/01/13/pushing-back-on-userland-hooks-with-cobalt-strike/" target="_blank">Pushing back on userland hooks with Cobalt Strike</a>. With all the direct syscall development recently, it was only a matter of time before Cobalt Strike got in on the fun. Mudge's <a href="https://github.com/rsmudge/unhook-bof" target="_blank">unhook-bof</a> adds a beacon command which will "refresh" the DLLs of the beacon process with unhooked variants.</li>
<li><a href="https://adepts.of0x.cc/shadowmove-hijack-socket/" target="_blank">Hijacking connections without injections: a ShadowMoving approach to the art of pivoting</a>. Have you ever wanted to use a legitimate process to send out your command and control traffic? By reusing existing network connections via the ancillary function driver that can be duplicated the <a href="https://www.usenix.org/system/files/sec20summer_niakanlahiji_prepub.pdf" target="_blank">ShadowMove</a> authors are able to send and receive messages over an existing socket without process injection or any special privileges.</li>
<li><a href="https://theevilbit.github.io/posts/divide_and_conquer/" target="_blank">Divide and Conquer - A technique to bypass NextGen AV</a>. Behavioral "netgen" antivirus and endpoint detection software often gives a process a maliciousness score based on how many "bad" actions it takes. The idea behind <a href="https://gist.github.com/theevilbit/073ca4eb15383eb3254272fc24632efd" target="_blank">divide and conquer</a> (and the earlier <a href="https://github.com/HexHive/malWASH" target="_blank">malWASH</a>) is to have different processes do each step of the malicious activity so no one process crosses the detection threshold of the AV/EDR product.</li>
<li><a href="https://www.mdsec.co.uk/2021/01/breaking-the-browser-a-tale-of-ipc-credentials-and-backdoors/" target="_blank">Breaking The Browser – A tale of IPC, credentials and backdoors</a>. As everyone and everything moves to a single sign on web app, the browsers have become the gatekeepers to valuable information. By injecting into Chrome, all kinds of tricks can be pulled - from credential stealing to a full implant stager.</li>
</ul>

<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/ajpc500/BOFs/tree/main/StaticSyscallsDump" target="_blank">StaticSyscallsDump</a> is a Beacon Object File (BOF) for Cobalt Strike that unhooks the NtReadVirtualMemory function and uses it do create a minidump of a process (i.e. lsass). This should be able to bypass many AV/EDR solutions, and works on a wide range of Windows versions unlike other syscall memory dumpers (i.e. <a href="https://github.com/b4rtik/SharpMiniDump/tree/master/SharpMiniDump" target="_blank">SharpMiniDump</a>)</li>
<li><a href="https://github.com/nccgroup/abstractshimmer" target="_blank">abstractshimmer</a>, the PoC for CVE-2020-15257 (containerd host network breakout from LWiS 2020-12-14) is out now. This could be a great breakout if you land on a somewhat privileged container.</li>
<li><a href="https://github.com/RCStep/CSSG" target="_blank">CSSG</a> is an aggressor and python script used to more easily generate and format beacon shellcode. Generates beacon stageless shellcode with exposed exit method, additional formatting, encryption, encoding, compression, multiline output, etc.</li>
<li><a href="https://github.com/moloch--/denim" target="_blank">Denim</a> makes compiling nim code for windows with obfuscator-llvm easy!</li>
<li><a href="https://github.com/RedSection/printjacker" target="_blank">printjacker</a> is a post-exploitation tool that creates a persistence mechanism by overwriting Printconfig.dll with a shellcode injector. The persistence mechanism can be invoked via executing wmic printer list command with any user. The shellcode will be executed with SYSTEM privileges. More details <a href="https://github.com/RedSection/printjacker/blob/main/POST.md" target="_blank">here</a>.</li>
<li><a href="https://github.com/mgeeky/UnhookMe" target="_blank">UnhookMe</a> is a universal Windows API resolver &amp; unhooker addressing problem of invoking unmonitored system calls from within of your Red Teams malware.</li>
</ul>
