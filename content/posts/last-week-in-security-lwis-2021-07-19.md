<p>Last Week in Security is a summary of the interesting cybersecurity news, techniques, tools and exploits from the previous week. This post covers 2021-07-12 to 2021-07-19.</p>
<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://www.theguardian.com/world/2021/jul/18/revealed-leak-uncovers-global-abuse-of-cyber-surveillance-weapon-nso-group-pegasus" target="_blank">Revealed: leak uncovers global abuse of cyber-surveillance weapon</a>. The latest model of iPhone on the latest version of iOS appears to be vulnerable to a 0click remote code execution exploit developed by NSO group and used to target political enemies and human rights activists around the world. While it isn't persistent, exploitation is reliable enough to simply re-exploit target devices. If you are an Android user feeling smug, rest assured there is a 0click RCE out there for your device as well. Stay updated and reboot your phones often! NSO group has denied any wrongdoing while releasing <a href="https://twitter.com/zackwhittaker/status/1417127080672776192" target="_blank">contradictory statements</a>. Details and IOCs <a href="https://www.amnesty.org/en/latest/research/2021/07/forensic-methodology-report-how-to-catch-nso-groups-pegasus/" target="_blank">here</a>, and a tool for analysis of backups or filesystem dumps <a href="https://github.com/mvt-project/mvt/" target="_blank">here</a>.</li>
<li><a href="https://citizenlab.ca/2021/07/hooking-candiru-another-mercenary-spyware-vendor-comes-into-focus/" target="_blank">Hooking Candiru Another Mercenary Spyware Vendor Comes into Focus</a>. More shady Israeli "cybersecurity" firms selling 0days that end up being used against political enemies. You've got to hand it to the <a href="https://www.startupnationcentral.org/" target="_blank">"startup nation,"</a> it's been churning out more 0day vendors (likely <a href="https://en.wikipedia.org/wiki/Unit_8200" target="_blank">unit 8200</a> "graduates") than any other country.</li>
<li><a href="https://blog.google/threat-analysis-group/how-we-protect-users-0-day-attacks/" target="_blank">How we protect users from 0-day attacks</a>. This post details three campaigns: emailed links, Office documents that loaded web content in IE, and LinkedIn messages with links to exploit 0days in Chrome and IE.</li>
<li><a href="https://www.state.gov/rewards-for-justice-reward-offer-for-information-on-foreign-malicious-cyber-activity-against-u-s-critical-infrastructure/" target="_blank">Rewards for Justice – Reward Offer for Information on Foreign Malicious Cyber Activity Against U.S. Critical Infrastructure</a>. The US government is offering rewards for tips that lead to the identification or location of any person who, while acting at the direction or under the control of a foreign government, participates in malicious cyber activities against U.S. critical infrastructure. They are running a SecureDrop instance on Tor as well. Maybe it is <a href="https://portswigger.net/daily-swig/revil-infrastructure-disappearance-sparks-speculation-about-fate-of-infamous-ransomware-slingers" target="_blank">already working</a>? An interesting note: "Reward payments may include payments in cryptocurrency." This is the first time, to my knowledge, the US federal government has offered to transact in cryptocurrency in any capacity.</li>
<li><a href="https://cloud.google.com/blog/products/identity-security/bringing-bimi-to-gmail-in-google-workspace" target="_blank">Advancing email security for Gmail and beyond with BIMI</a>. The new Brand Indicators for Message Identification (<a href="https://bimigroup.org/" target="_blank">BIMI</a>) is a standard for hosting images to be included as logos for emails. It requires a SPF, DKIM, a good DMARC policy, a special type of SVG, a special DNS entry, and for verification: the logo be trademarked, and then a "verified mark certificate" (VMC) must be purchased and hosted with the SVG. Hopefully Let's Encrypt will be a VMC issuer soon?</li>
<li><a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2021-34481" target="_blank">Windows Print Spooler Elevation of Privilege Vulnerability</a>. PrintNightmare refuses to die. Benjamin Delpy has been keeping the PoCs rolling, even creating a "<a href="https://twitter.com/gentilkiwi/status/1416429860566847490" target="_blank">privesc as a service</a>" hosting two printers that will write to System32 as SYSTEM. Don't worry, it works against <a href="https://twitter.com/gentilkiwi/status/1416190284216557570" target="_blank">non-domain joined Windows 11 machines</a> too. If you wrap buggy code from 1994 in virtualization based security, it's still buggy code. Confused on how this all works? Read this: <a href="https://thalpius.com/2021/07/16/windows-print-spooler-elevation-of-privilege-vulnerability-cve-2021-1675-explained/" target="_blank">Windows Print Spooler Elevation of Privilege vulnerability (CVE-2021-1675) explained</a>.</li>
<li><a href="https://ost2.fyi/" target="_blank">The new #OpenSecurityTraining2 site has been launched at ost2.fyi!</a>. Anyone can now sign up for the public betas of the first classes (with more to come soon!).</li>
<li><a href="https://twitter.com/jonasLyk/status/1417205166172950531" target="_blank">[DEVELOPING] Windows 10 feature upgrades leave SAM and SYSTEM hive readable by any user</a>. This has been seen on <a href="https://twitter.com/gentilkiwi/status/1417229454305267714" target="_blank">Windows 11</a> and <a href="https://twitter.com/mpgn_x64/status/1417243594663018499" target="_blank">fully up to date Windows 10</a>.</li>
</ul>
</section>
<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://alephsecurity.com/2021/07/15/aruba-instant/" target="_blank">Aruba in Chains: Chaining Vulnerabilities for Fun and Profit</a>. Finding "smaller" bugs and then chaining them together leads to full unauthenticated remote code execution against an Aruba router running Aruba Instant. PoC <a href="https://github.com/alephsecurity/research/blob/master/PoCs/Aruba/full_rce.py" target="_blank">here</a>.</li>
<li><a href="https://connormcgarr.github.io/swimming-in-the-kernel-pool-part-2/" target="_blank">Exploit Development: Swimming In The (Kernel) Pool - Leveraging Pool Vulnerabilities From Low-Integrity Exploits, Part 2</a>. Connor is back with another banger. Another full walk through with detailed steps, screenshots, and full code. If you are interested in exploit development for windows, this is a must read.</li>
<li><a href="https://blog.ryotak.me/post/cdnjs-remote-code-execution-en/" target="_blank">Remote code execution in cdnjs of Cloudflare</a>. A vulnerability in the way Cloudflare automatically updated its JavaScript CDN allowed for RCE. Props to Cloudflare for the near instant incident response and remediation - impressive.</li>
<li><a href="https://unit42.paloaltonetworks.com/single-bit-trap-flag-intel-cpu/" target="_blank">Evade Sandboxes With a Single Bit – the Trap Flag</a>. A relatively simple check can be used to determine if you are in a VM or on an actual host.</li>
<li><a href="https://williamknowles.io/fetching-sharphound-data-entirely-in-memory-no-dropped-zip-or-json-files-using-bof-net-and-cobalt-strike/" target="_blank">Fetching SharpHound data entirely in-memory (no dropped ZIP or JSON files) using BOF.NET and Cobalt Strike</a>. This is so good I had to test and implement it in my tooling the same day. Stop dropping files to target (even encrypted), and pull them straight back via BOF.NET. Be sure to check out <a href="https://github.com/anthemtotheego/CredBandit/commit/bb916de6ebea4e70b47664daf8304a2145c0abf5" target="_blank">this commit</a> to <a href="https://github.com/anthemtotheego/CredBandit" target="_blank">CredBandit</a> if you are interested in implementing this in your own BOFs (or just use the new <a href="https://github.com/CCob/BOF.NET#cobalt-strike-client-integrations" target="_blank">BOF.NET</a>). <a href="https://twitter.com/spotheplanet" target="_blank">@spotheplanet</a> has an <a href="https://www.ired.team/offensive-security/credential-access-and-credential-dumping/dumping-lsass-passwords-without-mimikatz-minidumpwritedump-av-signature-bypass#minidumpwritedump-to-memory-using-minidump-callbacks" target="_blank">updated section</a> on dumping to memory as well.</li>
<li><a href="https://www.romainthomas.fr/post/21-07-pokemongo-anti-frida-jailbreak-bypass/" target="_blank">Gotta Catch 'Em All: Frida &amp; jailbreak detection</a>. If you have any interest in iOS security or jailbreak detection, this post is full of great details.</li>
</ul>
</section>
<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/synacktiv/CVE-2021-3492" target="_blank">CVE-2021-3492</a> is an exploit in the shiftfs driver in Ubuntu that was introduced in April 2019, affecting at least 20.04 and 20.10. It was used in Pwn2Own successfully, with the full details released this week in a <a href="https://www.synacktiv.com/en/publications/exploitation-of-a-double-free-vulnerability-in-ubuntu-shiftfs-driver-cve-2021-3492.html" target="_blank">blog post</a>.</li>
<li><a href="https://github.com/S3cur3Th1sSh1t/SharpImpersonation" target="_blank">SharpImpersonation</a> is a token impersonation tool written in C#. Lots of details in this <a href="https://s3cur3th1ssh1t.github.io/SharpImpersonation-Introduction/" target="_blank">blog post</a>.</li>
<li><a href="https://github.com/OG-Sadpanda/SharpExcelibur" target="_blank">SharpExcelibur</a> is a tool to read Excel spreadsheets (XLS/XLSX) using Cobalt Strike's execute-assembly functionality.</li>
<li><a href="https://github.com/boku7/injectAmsiBypass" target="_blank">injectAmsiBypass</a> is a Cobalt Strike Beacon Object File (BOF) that bypasses AMSI in a remote process with code injection.</li>
<li><a href="https://github.com/topotam/PetitPotam" target="_blank">PetitPotam</a> is a PoC tool to coerce Windows hosts to authenticate to other machines via MS-EFSRPC EfsRpcOpenFileRaw function. Disabling the EFS service seems not to mitigate the "feature".</li>
<li><a href="https://github.com/klezVirus/CheeseTools#cheesesql" target="_blank">CheeseSQL</a> is Command Exec / Lateral Movement via MSSQL Trust. This tool has been developed to overcome some of the limitations given by already existing tools like <a href="https://github.com/NetSPI/ESC" target="_blank">ESC</a>, mostly regarding MSSQL impersonation. Moreover, CheeseSQL has been specifically modified to run from Covenant (via reflective loading), and to automate the most important phases of MSSQL trust abuse.</li>
</ul>
