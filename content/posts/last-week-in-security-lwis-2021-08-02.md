<p>Last Week in Security is a summary of the interesting cybersecurity news, techniques, tools and exploits from the previous week. This post covers 2021-07-26 to 2021-08-02.</p>
<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://bughunters.google.com/learn" target="_blank">Welcome to Bug Hunter University</a>. Google launches their own educational content aimed at bug hunters that are working on Google products. Don't expect technique walkthroughs, this is more of a detailed guide on how to bug hunt against Google (i.e. what is in scope, what is considered auth bypass, etc).</li>
<li><a href="https://www.cisa.gov/blog/2021/07/29/cisa-announces-new-vulnerability-disclosure-policy-vdp-platform" target="_blank">CISA Announces New Vulnerability Disclosure Policy (VDP) Platform</a>. The US federal government is getting into the bug bounty game with the help of BugCrowd. The Department of Homeland Security (DHS), the Department of Labor (DoL), and the Department of Interior (DoI) are among the agencies planning to leverage this platform at the onset.</li>
<li><a href="https://edition.cnn.com/2021/07/30/tech/amazon-eu-privacy-fine/index.html" target="_blank">Amazon hit by record $887 million EU privacy fine</a>. The EU says Amazon processed personal data in ways that violated GDPR requirements, Amazon said the decision was "without merit." Looks like the real winners in this case will be the lawyers.</li>
<li><a href="https://twitter.com/domchell/status/1421148920340701192" target="_blank">MDSec pushes Nighthawk C2 framework PR via Twitter</a>. The upcoming commercial C2 from MDSec looks like it has some pretty interesting features: hot swappable C2 profiles, in memory encryption for evasion, BOF compatibility, etc. "Coming soon."</li>
<li><a href="https://posts.specterops.io/introducing-bloodhound-enterprise-attack-path-management-for-everyone-39cfd8d6eb7c" target="_blank">Introducing BloodHound Enterprise: Attack Path Management for Everyone</a>. The enterprise version of the extremely popular BloodHound tool is out now! If you have a massive AD environment, it is likely worth the cost to get what amounts to a top tier AD penetration test with helpful interactive remediation and retesting.</li>
<li><a href="https://portswigger.net/web-security/certification" target="_blank">PortSwigger launches Burp Suite Certified Practitioner</a>. All the training material and even a practice exam are available for free, and the cost is very reasonable at $99. The certification expires after five years with no word on if you have to pay to "maintain" it beyond that time.</li>
</ul>

<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://dirkjanm.io/ntlm-relaying-to-ad-certificate-services/" target="_blank">NTLM relaying to AD CS - On certificates, printers and a little hippo</a>. The AD GOAT is back to lay it down on NTLM relaying, and even add a little bit of his own twist with <a href="https://github.com/dirkjanm/PKINITtools" target="_blank">PKINITtools</a>. If you only read one post about the latest AD CS relaying and PetitPotam, read this one. Want to use Cobalt Strike for this? Read <a href="https://rastamouse.me/ntlm-relaying-via-cobalt-strike/" target="_blank">NTLM Relaying via Cobalt Strike</a>.</li>
<li><a href="https://dozer.nz/posts/CVE-2020-36239-POC-dev" target="_blank">Developing an exploit for the Jira Data Center Ehcache RCE (CVE-2020-36239)</a>. I love this kind of post. It walks through every step from reading a bug advisory to RCE and all the struggles, blog posts, and different attempts along the way.</li>
<li><a href="https://dolosgroup.io/blog/2021/7/9/from-stolen-laptop-to-inside-the-company-network" target="_blank">From Stolen Laptop to Inside the Company Network</a>. Think your BitLocker encrypted laptop is safe from a determined adversary? Think again. The trusted platform module (TPM) sends the BitLocker encryption key via Serial Peripheral Interface (SPI) in plaintext. A bit of research and a quick hookup with <a href="https://www.saleae.com/" target="_blank">Saleae</a> spill the beans. The SSD was then extracted and decrypted. Because the target had a "pre-logon" VPN tunnel setup, the assessors were able to build a test VM and connect to internal file shares. Very nice work against a hardened laptop. Enable that pre-boot authentication!</li>
<li><a href="https://www.solomonsklash.io/stealing-tokens-with-malicious-driver.html" target="_blank">Stealing Tokens In Kernel Mode With A Malicious Driver</a>. This post walks through building a simple driver to copy access tokens between PIDs to allow user spoofing or privilege escalation. Bypassing driver signing is another topic all together, but the basics of kernel development and userspace to kernel communication are covered here nicely.</li>
<li><a href="https://voidsec.com/root-cause-analysis-of-cve-2021-3438/" target="_blank">Root Cause Analysis of a Printer’s Drivers Vulnerability CVE-2021-3438</a>. Last week's SSPORT.sys printer driver vulnerability may have been oversold! VoidSec breaks down the root cause and describes why it can be, at best, a denial of service exploit.</li>
<li><a href="https://saaramar.github.io/IOMobileFrameBuffer_LPE_POC/" target="_blank">WebContent-&gt;EL1 LPE: OOBR in AppleCLCD / IOMobileFrameBuffer</a>. If nothing else, this is good proof of "parallel discovery" even against a "hard target" like iOS. The <a href="https://github.com/saaramar/IOMobileFrameBuffer_LPE_POC" target="_blank">POC</a> is available, but without the arbitrary read/write needed to finish it.</li>
<li><a href="https://itm4n.github.io/fuzzing-windows-rpc-rpcview/" target="_blank">Fuzzing Windows RPC with RpcView</a> introduces the process to enumerate RPC servers with RpCView. Expect some good stuff from itm4n as a result of this.</li>
<li><a href="https://klezvirus.github.io/RedTeaming/AV_Evasion/CodeExeNewDotNet/" target="_blank">The path to code execution in the era of EDR, Next-Gen AVs, and AMSI</a> introduces <a href="https://github.com/klezVirus/inceptor" target="_blank">inceptor</a>, a template-based PE packer for Windows, designed to help penetration testers and red teamers to bypass common AV and EDR solutions. Inceptor has been designed with a focus on usability, and to allow extensive user customization. Inceptor is a framework that wraps many other useful tools, sgn, sRDI, donut, DInvoke, Syswhispers, ConfuserEx, Chameleon, LLVM-Obfuscator, and others to create an easy to use tool chain to wrap, compile, and obfuscate input shellcode or PE files. This could be a very useful base to extend with private templates and incorporate into your own workflow.</li>
<li><a href="https://pentestlab.blog/2021/08/02/universal-privilege-escalation-and-persistence-printer/" target="_blank">Universal Privilege Escalation and Persistence – Printer</a>. The PrintNightmare saga may have cooled off, but this post explores how to set up your own rogue printer for that double-click to system privilege escalation.</li>
</ul>

<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/AzAgarampur/byeintegrity8-uac" target="_blank">byeintegrity8-uac</a> is a Windows 7 to Windows 11 compatible "Always Notify" UAC bypass. It's also been implemented in <a href="https://github.com/hfiref0x/UACME/issues/111" target="_blank">UACME</a> as technique #69.</li>
<li><a href="https://bugs.chromium.org/p/project-zero/issues/detail?id=2186" target="_blank">Issue 2186: Exchange: AD Schema Misconfiguration Elevation of Privilege</a>. Installing Exchange in an AD environment modified the AD schema in a way that allowed computer accounts to create arbitrary AD objects as children (users, etc). This was patched in the Exchange cumulative updates release on 2021-06-29 but is worth checking for on your next assessment.</li>
<li><a href="https://blog.cobaltstrike.com/2021/07/29/introducing-mimikatz-kit/" target="_blank">Introducing Mimikatz Kit</a>. HelpSystems has decoupled Mimikatz from CobaltStrike releases with Mimikatz Kit. With the rapid rate of new features in Mimikatz recently this is a welcome change.</li>
<li><a href="https://github.com/DigeeX/raider" target="_blank">raider</a> is a framework designed to test authentication for web applications. While web proxies like ZAProxy and Burpsuite allow authenticated tests, they don't provide features to test the authentication process itself, i.e. manipulating the relevant input fields to identify broken authentication. Most authentication bugs in the wild have been found by manually testing it or writing custom scripts that replicate the behaviour. Raider aims to make testing easier, by providing the interface to interact with all important elements found in modern authentication systems. It uses a Lisp like configuration language to control the authentication flows.</li>
<li><a href="https://github.com/bats3c/ADCSPwn" target="_blank">ADCSPwn</a> is a tool to escalate privileges in an active directory network by coercing authenticate from machine accounts (Petitpotam) and relaying to the certificate service. This is your easy button for PetitPotam + ESC8 exploitation.</li>
<li><a href="https://shells.systems/ninjac2-v2-1-new-webshell-agent-more-features-and-updated-av-bypass/" target="_blank">NinjaC2 V2.1 : New webshell agent, more features and updated AV bypass</a>. The update adds a webshell and a few other AV bypass features.</li>
<li><a href="https://github.com/chompie1337/Linux_LPE_eBPF_CVE-2021-3490" target="_blank">Linux_LPE_eBPF_CVE-2021-3490</a> is an LPE exploit for CVE-2021-3490. Tested on Ubuntu 20.10 (Groovy Gorilla) kernels 5.8.0-25.26 through 5.8.0-52.58. and Ubuntu 21.04 (Hirsute Hippo) 5.11.0-16.17. Full details in <a href="https://www.graplsecurity.com/post/kernel-pwning-with-ebpf-a-love-story" target="_blank">Kernel Pwning with eBPF: a Love Story</a>.</li>
<li><a href="https://github.com/ShutdownRepo/pywhisker#import-and-export" target="_blank">pywhisker</a> is a Python equivalent of the original <a href="https://github.com/eladshamir/Whisker" target="_blank">Whisker</a> made by Elad Shamir and written in C#. This tool allows users to manipulate the msDS-KeyCredentialLink attribute of a target user/computer to obtain full control over that object. It's based on Impacket and on our Python equivalent of Michael Grafnetter's DSInternals called PyDSInternals. This tool, along with Dirk-jan's PKINITtools allow for a complete primitive exploitation on UNIX-based systems only.</li>
<li><a href="https://github.com/ShutdownRepo/targetedKerberoast" target="_blank">targetedKerberoast</a> is a Python script that can, like many others (e.g. GetUserSPNs.py), print "kerberoast" hashes for user accounts that have a SPN set. This tool brings the following additional feature: for each user without SPNs, it tries to set one (abuse of a write permission on the servicePrincipalName attribute), print the "kerberoast" hash, and delete the temporary SPN set for that operation. This is called targeted Kerberoasting. This tool can be used against all users of a domain, or supplied in a list, or one user supplied in the CLI.</li>
<li><a href="https://github.com/kyleavery/scarecrow_wrapper" target="_blank">scarecrow_wrapper</a> is wrapper payload for Mythic that wraps any agent shellcode with the <a href="https://github.com/optiv/ScareCrow" target="_blank">ScareCrow</a> loader. This wrapper currently supports CPL, EXE, and DLL payload types from ScareCrow.</li>
<li><a href="https://github.com/cfalta/MicrosoftWontFixList/blob/main/README.md" target="_blank">MicrosoftWontFixList</a>. Are you lost in all the "Won't fix" vulnerabilities released or discovered in July? This page has them all summarized for you.</li>
<li><a href="https://github.com/boku7/spawn" target="_blank">spawn</a> is a Cobalt Strike BOF that spawns a sacrificial process, injects it with shellcode, and executes payload. Built to evade EDR/UserLand hooks by spawning sacrificial process with Arbitrary Code Guard (ACG), BlockDll, and PPID spoofing.</li>
<li><a href="https://github.com/SySS-Research/hallucinate" target="_blank">hallucinate</a> is a one-stop TLS traffic inspection and manipulation using dynamic instrumentation. For more information check out the introductory <a href="https://blog.syss.com/posts/hallucinate/" target="_blank">blog post</a>.</li>
<li><a href="https://github.com/tnpitsecurity/ligolo-ng" target="_blank">ligolo-ng</a> is an advanced, yet simple, tunneling/pivoting tool that uses a TUN interface. Instead of using a SOCKS proxy or TCP/UDP forwarders, Ligolo-ng creates a userland network stack using Gvisor.</li>
<li><a href="https://github.com/mxrch/revealin" target="_blank">revealin</a> is a tool to uncover the full name of a target on Linkedin by taking advantage of the autocomplete feature.</li>
</ul>
