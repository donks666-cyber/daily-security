<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://www.vusec.net/projects/blindside/" target="_blank">BlindSide</a> allows attackers to “hack blind” in the Spectre era. That is, given a simple buffer overflow in the kernel and no additional info leak vulnerability, BlindSide can mount BROP-style attacks in the speculative execution domain to repeatedly probe and derandomize the kernel address space, craft arbitrary memory read gadgets, and enable reliable exploitation. Go from BoF to root in ~200 seconds. Demo <a href="https://youtu.be/m-FUIZiRN5o" target="_blank">here</a>.</li>
<li><a href="https://byte.how/posts/collaborative-reverse-engineering/" target="_blank">Collaborative Reverse Engineering with Ghidra Server</a>. Ghidra has built in git for collaboration, but it can be hard to setup and use. This post introduces <a href="https://github.com/bytehow/docker-ghidra-server" target="_blank">docker-ghidra-server</a> and shows how it can be used to work on reverse engineering projects as part of a team.</li>
<li><a href="https://www.secjuice.com/advanced-sqli-waf-bypass/" target="_blank">Advanced boolean-based SQLi filter bypass techniques</a>. This article discusses bypasses for <a href="https://github.com/client9/libinjection" target="_blank">libinjection</a>, a popular library used in web application firewalls (WAFs) to detect and prevent SQL injection attacks.</li>
<li><a href="https://malicious.link/post/2020/run-as-system-using-evil-winrm/" target="_blank">Run as SYSTEM using Evil-WinRM</a>. With PSExec being heavily monitored, this short post shows how to use Evil-WinRM to execute as SYSTEM remotely with powershell remoteing instead.</li>
<li><a href="https://s3cur3th1ssh1t.github.io/Bypass-AMSI-by-manual-modification-part-II/" target="_blank">Bypass AMSI by manual modification part II - Invoke-Mimikatz</a>. This post goes into detail into how Invoke-Mimikatz works, then makes a bunch of modifications to defeat static AMSI triggers. Defender runtime analysis defeat is left as an exercise for the reader.</li>
<li><a href="https://github.com/mdsecresearch/Publications/blob/master/presentations/Offensive%20Development%20-%20Post-Exploitation%20Tradecraft%20in%20an%20EDR%20World%20-%20x33fcon%202020.pdf" target="_blank">Offensive Development - Post-Exploitation Tradecraft in an EDR World</a>. This presentation by Dominic Chell shows how DevOps principles can be used to help red teams operate in EDR-laden environments and avoid detection. Excellent work here.</li>
<li><a href="https://blog.fox-it.com/2020/09/10/streamdivert-relaying-specific-network-connections/" target="_blank">StreamDivert: Relaying (specific) network connections</a> Written for a case where a server could issue commands to clients, but the service was fragile and couldn't be shut down, StreamDivert uses a driver on Windows to selectively re-route packets based on a filter statement. Escape the DMZ with this. Tool <a href="https://github.com/jellever/StreamDivert" target="_blank">here</a>.</li>
<li><a href="https://slaeryan.github.io/posts/midnighttrain.html" target="_blank">Introducing MIDNIGHTTRAIN - A Covert Stage-3 Persistence Framework weaponizing UEFI variables</a>. Slaeryan shows the power of being well read in cybersecurity and having the ability to combine research into novel and powerful tools - in this case some really cool Windows persistence. This post is long, but worth it.</li>
<li><a href="https://blog.orange.tw/2020/09/how-i-hacked-facebook-again-mobileiron-mdm-rce.html" target="_blank">How I Hacked Facebook Again! Unauthenticated RCE on MobileIron MDM</a>. Orange Tsai is a legend, and it has been almost a year since he pwned <a href="https://blog.orange.tw/2019/07/attacking-ssl-vpn-part-1-preauth-rce-on-palo-alto.html" target="_blank">many</a> <a href="https://blog.orange.tw/2019/08/attacking-ssl-vpn-part-2-breaking-the-fortigate-ssl-vpn.html" target="_blank">major</a> <a href="https://blog.orange.tw/2019/09/attacking-ssl-vpn-part-3-golden-pulse-secure-rce-chain.html" target="_blank">VPNs</a>. This time he goes after the MobileIron MDM and combines 3 CVEs to get a shell on Facebook infrastructure. Impressive as always - PoC <a href="https://github.com/iamnoooob/CVE-Reverse/tree/master/CVE-2020-15505" target="_blank">here</a>.</li>
</ul>

<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://www.secura.com/pathtoimg.php?id=2055" target="_blank">Zerologon: Unauthenticated domain controller compromise by subverting Netlogon cryptography (CVE-2020-1472)</a>. A holdover feature from Windows NT that allows computer accounts to authenticate to a Domain Controller via NetLogon and uses AES-CFB8 with a zero'd IV means that 1/256 attempts with a zero'd client challenge (attacker controlled) will result in a session key of all zeros. Since computer accounts don't get locked out, an attacker can authenticate to your Domain Controller as the Domain Controller computer account in under 256 tries. From there, a password reset can be issued and all hashes dumped. With these hashes, a pass-the-hash (or golden ticket) can be used to log back into the DC and reset its computer password back to whatever it was before the attack. This is the worst bug since MS17-010 aka ETERNALBLUE and I predict it will be used in all types of attacks but ransomware just got an "easy button" for complete domain compromise. This is a drop-everything-and-patch scenario. <a href="https://github.com/blackarrowsec/redteam-research/tree/master/CVE-2020-1472" target="_blank">Multiple</a> <a href="https://github.com/dirkjanm/CVE-2020-1472/blob/master/cve-2020-1472-exploit.py" target="_blank">PoCs</a> <a href="https://github.com/dirkjanm/CVE-2020-1472" target="_blank">exist (this one is the best)</a>. A <a href="https://tdm.socprime.com/tdm/info/FgNYLnTxIVrs/7WbXfnQBSh4W_EKGaxL5/" target="_blank">Sigma rule</a> is also available (requires registration with SOCPrime but the rule is free).</li>
<li><a href="https://www.gosecure.net/blog/2020/09/08/wsus-attacks-part-2-cve-2020-1013-a-windows-10-local-privilege-escalation-1-day/" target="_blank">WSUS Attacks Part 2: CVE-2020-1013 a Windows 10 Local Privilege Escalation 1-Day</a>. The ability to intercept WSUS locally, and add a certificate to the current user's local store means that arbitrary Microsoft signed (psexec) binaries can be run as SYSTEM by any user. Tool release coming in less than 30 days.</li>
<li><a href="https://breakdev.org/evilginx-2-4-gone-phishing/" target="_blank">Evilginx 2.4 - Gone Phishing</a>. Evilginx is an amazing tool for phishing assessments, and it just got better. Kuba added a bunch of neat new features like pre-load pages and IP blacklisting. Be sure to update and try them out!</li>
<li><a href="https://github.com/alessandrod/snuffy" target="_blank">snuffy</a> is a simple command line tool to dump the data sent and received by programs that use OpenSSL. <a href="https://confused.ai/posts/intercepting-zoom-tls-encryption-bpf-uprobes" target="_blank">Here</a> is an example of dumping data from Zoom.</li>
<li><a href="https://srcincite.io/advisories/src-2020-0019/" target="_blank">SRC-2020-0019 : Microsoft Exchange Server DlpUtils AddTenantDlpPolicy Remote Code Execution Vulnerability</a>. This is an authenticated RCE against Exchange servers that allows any authenticated user to execute code as SYSTEM! Both <a href="https://srcincite.io/pocs/cve-2020-16875.py.txt" target="_blank">HTTPS</a> and <a href="https://srcincite.io/pocs/cve-2020-16875.ps1.txt" target="_blank">Ps-Remoting</a> PoCs are available - patch now!</li>
<li><a href="https://bugs.chromium.org/p/project-zero/issues/detail?id=2051" target="_blank">Windows: CloudExperienceHostBroker Unsafe COM Object EoP</a>. COM objects with bad access control lists (ACLs) lead to the ability to add a user as an Administrator from any account. C# PoC <a href="https://bugs.chromium.org/p/project-zero/issues/attachment?aid=449687&amp;signed_aid=Zh89DlsWY5cj_V-HXxphCQ==" target="_blank">here</a>.</li>
<li><a href="https://github.com/JuxhinDB/twistrs" target="_blank">twistrs</a> is a domain name permutation and enumeration library powered by Rust. It most cases it has better coverage and is much faster than <a href="https://github.com/elceef/dnstwist" target="_blank">dnstwist</a>. Blog post <a href="https://blog.digital-horror.com/twistrs/" target="_blank">here</a>.</li>
</ul>
