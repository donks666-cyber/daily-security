<section id="news">
<h2>News</h2>
<ul>
<li><dl>
<dt>Solargate aka Solorigate aka the SolarWinds compromise fallout</dt>
<dd><ul>
<li><a href="https://www.microsoft.com/security/blog/2020/12/18/analyzing-solorigate-the-compromised-dll-file-that-started-a-sophisticated-cyberattack-and-how-microsoft-defender-helps-protect/" target="_blank">Analyzing Solorigate, the compromised DLL file that started a sophisticated cyberattack, and how Microsoft Defender helps protect customers</a>. This post from Microsoft details the major parts of the attack. I am a bit surprised that after all the work to backdoor the software supply chain and remain undetected, the actors used PowerShell for post exploitation activities instead of in memory C#.</li>
<li><a href="https://blog.truesec.com/2020/12/17/the-solarwinds-orion-sunburst-supply-chain-attack/" target="_blank">The SolarWinds Orion SUNBURST supply-chain Attack</a>. Using passive DNS records from <a href="https://oscontext.com/" target="_blank">Open Source Context</a>, the team at truesec manage to shed some light on the victims by <a href="https://github.com/RedDrip7/SunBurst_DGA_Decode/blob/main/decode.py" target="_blank">decoding</a> the generated DNS lookups. Cloudflare <a href="https://blog.cloudflare.com/a-quirk-in-the-sunburst-dga-algorithm/" target="_blank">has more details</a>.</li>
<li><a href="https://www.trustedsec.com/blog/solarwinds-backdoor-sunburst-incident-response-playbook/" target="_blank">SolarWinds Backdoor (Sunburst) Incident Response Playbook</a>. This is a no nonsense playbook to follow if you are unfortunate enough to have a SolarWinds deployment.</li>
<li><a href="https://unit42.paloaltonetworks.com/solarstorm-supernova/" target="_blank">SUPERNOVA: A Novel .NET Webshell</a>. One post exploitation technique implemented was a novel .NET webshell that showed advanced tradecraft with in-memory execution.</li>
<li><a href="https://gist.github.com/0xsha/75616ef6f24067c4fb5b320c5dfa4965" target="_blank">CVE-2020-10148 (local file disclosure PoC for SolarWinds Orion aka door to SuperNova?)</a>. Once SolarWinds Orion hit the news, researches started to look into it and like most enterprise apps, CVEs fell out. Description <a href="https://kb.cert.org/vuls/id/843464" target="_blank">here</a>.</li>
<li><a href="https://www.fireeye.com/blog/threat-research/2020/12/sunburst-additional-technical-details.html" target="_blank">SUNBURST Additional Technical Details</a> is another great post from FireEye with more technical information about the attack. To crack the hashes of the AV strings the backdoor checks for yourself, a custom <a href="https://github.com/matrix/hashcat-SolarWindsHack" target="_blank">hashcat-SolarWindsHack</a> was used. All hashes have been cracked and are available <a href="https://github.com/fireeye/sunburst_countermeasures/blob/main/fnv1a_xor_hashes.txt" target="_blank">here</a>.</li>
<li><a href="https://msrc-blog.microsoft.com/2020/12/31/microsoft-internal-solorigate-investigation-update/" target="_blank">Microsoft Internal Solorigate Investigation Update</a>. Just when I thought the Solorigate news was drying up, Microsoft reveals that actors related to the breach had "been used to view source code in a number of source code repositories." Microsoft claims that no source was modified and any source disclosure would not impact security.</li>
</ul>
</dd>
</dl>
</li>
<li><a href="https://www.washingtonpost.com/technology/2020/12/29/apple-corellium-lawsuit/" target="_blank">Apple loses copyright battle against security start-up Corellium</a>. Apple once was in talks to buy the iOS emulation startup Corellium, but resorted to attempting to sue it out of existence instead. This was a major case for anyone doing security research, as a decision the other way would have major implications for anyone virtualizing or emulating software or hardware. This is a rare win for a great innovative startup against a mega-corp. Full decision <a href="https://www.courtlistener.com/docket/16064642/784/apple-inc-v-corellium-llc/" target="_blank">here</a>.</li>
<li><a href="https://citizenlab.ca/2020/12/the-great-ipwn-journalists-hacked-with-suspected-nso-group-imessage-zero-click-exploit/" target="_blank">The Great iPwn Journalists Hacked with Suspected NSO Group iMessage ‘Zero-Click’ Exploit</a>. This is the first major "in the wild" iOS exploit chain since <a href="https://www.zdnet.com/article/trident-ios-flaws-researchers-detail-how-the-spyware-stayed-hidden/" target="_blank">Trident</a> (2016). It used an no-click iMessages 0day that worked on the latest iPhone and iOS version (iPhone 11 on 13.5.1 at the time of the discovery). Much like Trident, this malware was used to target journalists with connections to the Middle East.</li>
<li><a href="https://github.com/checkra1n/pongoOS" target="_blank">pongoOS</a>. Speaking of iOS exploits, the checkra1n team has open sourced the entirety of their pongoOS, including kernel patchfinder and SEP exploit.</li>
<li><a href="https://www.eyecontrol.nl/blog/undocumented-user-account-in-zyxel-products.html" target="_blank">Undocumented user account in Zyxel products (CVE-2020-29583)</a>. "Undocumented user account" is a polite way to say backdoor. In this case the account was used by other Zyxel products to install firmware updates via FTP. If your update mechanism solution is to hard code admin credentials in all your devices, maybe you shouldn't be making security hardware?</li>
<li><a href="https://www.engadget.com/ticketmaster-pays-10-million-fine-to-avoid-prosecution-for-hacking-rival-110109597.html" target="_blank">Ticketmaster pays $10 million fine after hacking a startup rival</a>. After being hired to Ticketmaster from rival Songkick, an employee shared Songkick credentials and URLs with Ticketmaster which the company used to access its rival. While the $10 million fine seems light for the blatant "hacking" (they even held a "summit" where they gathered to use the credentials), karma may have dealt the true fine, with Ticketmaster's revenue down 95% year over year.</li>
<li><a href="https://berthub.eu/articles/posts/reverse-engineering-source-code-of-the-biontech-pfizer-vaccine/" target="_blank">Reverse Engineering the source code of the BioNTech/Pfizer SARS-CoV-2 Vaccine</a>. Not really security related but the way the vaccine is broken down in computer terms is really interesting.</li>
</ul>

<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://s3cur3th1ssh1t.github.io/Phish-password-protected-Excel-files/" target="_blank">Excel-Phish - Phish protected Excel-file passwords</a>. Sometimes the easiest way to crack the password of Excel documents is to ask for them. The trusted location allowing macros without warnings was key to pulling this off. Code <a href="https://github.com/S3cur3Th1sSh1t/Excel-Phish" target="_blank">here</a>.</li>
<li><a href="https://labs.ioactive.com/2020/12/warcodes-ii-desko-case.html" target="_blank">Warcodes II - The Desko Case</a>. Barcode programming has been <a href="https://github.com/UberKitten/barcOwned" target="_blank">demonstrated before</a> but any traveler likely has direct experience with these Desko scanners. Imagine what would happen if you reprogrammed a scanner to accept the next person in line regardless of ticket status.</li>
<li><a href="https://posts.specterops.io/hands-in-the-cookie-jar-dumping-cookies-with-chromiums-remote-debugger-port-34c4f468844e" target="_blank">Hands in the Cookie Jar: Dumping Cookies with Chromium’s Remote Debugger Port</a>. Cookie dumping is a very useful technique for modern red teams. As the use of multi-factor authentication increases, the ability to dump a cookie and browse to a site already authenticated is an attractive technique for adversaries. This <a href="https://github.com/slyd0g/WhiteChocolateMacademiaNut" target="_blank">new tool</a> is based on <a href="https://mango.pdf.zone/stealing-chrome-cookies-without-a-password" target="_blank">some older research</a>. I personally use <a href="https://github.com/djhohnstein/SharpChromium" target="_blank">SharpChromium</a>, but this new cross platform solution is a welcome addition!</li>
<li><a href="https://staaldraad.github.io/post/2020-12-15-cve-2020-25695-postgresql-privesc/" target="_blank">CVE-2020-25695 Privilege Escalation in Postgresql</a>. This is a great post as it shows the full process from looking at the source code to working exploit, and everything in between. The exploit is for PostgreSQL 11.9 to 13.0 and allows an unprivileged Postgres user to elevate to superuser within PostgreSQL.</li>
<li><a href="https://bad-jubies.github.io/Blind-SQLi-1/" target="_blank">Automating Blind Sql Injection</a> is a basic post on how to automate a basic blind SQLi in the popular <a href="https://github.com/digininja/DVWA" target="_blank">Damn Vulnerable Web App (DVWA)</a>. Lots of testers will throw <a href="https://github.com/sqlmapproject/sqlmap" target="_blank">SQLMap</a> without fully understanding how it works under the hood. This post is a good way to understand the basics before using more advanced tools.</li>
<li><a href="https://arxiv.org/pdf/2012.06884.pdf" target="_blank">AIR-FI: Generating Covert Wi-Fi Signals fromAir-Gapped Computers</a>. I can't see myself ever needing to use this technique but dang is it cool. An attacker can control RAM to emit 2.4 GHz signals that can be picked up by nearby WiFi enabled devices. Demo <a href="https://www.youtube.com/watch?v=vhNnc0ln63c" target="_blank">here</a>.</li>
<li><a href="https://fortynorthsecurity.com/blog/fastly-and-fronting/" target="_blank">Fastly and Fronting</a>. Domain fronting is not a new technique, and while it has suffered from dropped support by the big CDNs, it still works with Azure, Fastly, and others. This post shows you how to set it up from registering a service to testing the front. For a list of frontable domains, check out <a href="https://github.com/rvrsh3ll/FindFrontableDomains" target="_blank">FindFrontableDomains</a> or try <a href="https://censys.io/ipv4?q=autonomous_system.description.raw%3A+%22FASTLY%22" target="_blank">this Censys query</a> (credit to EX on the BloodHoundGang Slack for the query).</li>
<li><a href="https://www.n00py.io/2020/12/dumping-laps-passwords-from-linux/" target="_blank">Dumping LAPS Passwords from Linux</a>. Using a little bit of python (and the ldap3 module) or <cite>ldapsearch</cite> a Linux machine that can communicate with a DC and a user that has credentials to see the "Local Admin Password Solution" (LAPS) attribute in the DC (by default just domain admins) can dump all local admin passwords. Issue arise when misconfigurations allow non-DAs to see this attribute (help desk, etc) and their account is compromised.</li>
<li><a href="https://googleprojectzero.blogspot.com/2020/12/an-ios-hacker-tries-android.html" target="_blank">An iOS hacker tries Android</a>. Brandon Azad drops one last gem before moving from Google's Project Zero to Apple. In this low level post, he shows his work as he exploits an Android Neural Processing Unit bug. At the end he provides unique insight into the differences and similarities between Android and iOS kernel exploitation that he is uniquely qualified to present.</li>
<li><a href="https://redteamer.tips/a-tale-of-net-assemblies-cobalt-strike-size-constraints-and-reflection/?utm_source=rss&amp;utm_medium=rss&amp;utm_campaign=a-tale-of-net-assemblies-cobalt-strike-size-constraints-and-reflection" target="_blank">A tale of .NET assemblies, cobalt strike size constraints, and reflection.</a>. With Cobalt Strike's 1MB limit for <cite>execute-assembly</cite>, the ability to load dependancies at runtime and add them to the current AppDomain vs having to weave them into the binary allows you to remain under the 1MB limit. Rasta Mouse has a post on the same subject <a href="https://offensivedefence.co.uk/posts/assembly-resolve/" target="_blank">here</a></li>
<li><a href="https://x41-dsec.de/security/advisory/exploit/research/2020/12/21/x41-microsoft-exchange-rce-dlp-bypass/" target="_blank">Microsoft Exchange Remote Code Execution - CVE-2020-16875</a>. September saw this Exchange RCE patched, and now it is once again patched in December 2020. Say it again for the people in the back, "a patch is not always the death of a bug!"</li>
<li><a href="https://www.varonis.com/blog/exploiting-bginfo-to-infiltrate-a-corporate-network/" target="_blank">Exploiting BGInfo to Infiltrate a Corporate Network</a>. This is a really cool technique that exploits the program that generates the information overlays on Windows desktop backgrounds.</li>
<li><a href="https://outflank.nl/blog/2020/12/26/direct-syscalls-in-beacon-object-files/" target="_blank">Direct Syscalls in Beacon Object Files</a>. The OGs of direct syscalls are back with direct syscalls in Cobalt Strike BOFs. They drop <a href="https://github.com/outflanknl/WdToggle" target="_blank">WdToggle</a> and a helper tool <a href="https://github.com/outflanknl/InlineWhispers" target="_blank">InlineWhispers</a> which can help add direct syscalls to your BOFs. Another new project <a href="https://github.com/ajpc500/BOFs/tree/main/SyscallsInject" target="_blank">SyscallsInject</a> builds on this by reading the syscalls from on disk ntdll vs statically compiled into the BOF which I am calling "diet Hell's Gate" (true Hell's Gate would read the syscalls from memory).</li>
<li><a href="https://www.mdsec.co.uk/2020/12/bypassing-user-mode-hooks-and-direct-invocation-of-system-calls-for-red-teams/" target="_blank">Bypassing User-Mode Hooks and Direct Invocation of System Calls for Red Teams</a>. This is a round up of most (all?) of the ways to bypass user-mode hooks and use direct syscalls on Windows. Lots of good info here, and a new method I was unfamiliar with as well - <a href="https://github.com/crummie5/FreshyCalls" target="_blank">FreshyCalls</a>.</li>
<li><a href="https://s3cur3th1ssh1t.github.io/Playing-with-OffensiveNim/" target="_blank">Bypassing Windows protection mechanisms &amp; Playing with OffensiveNim</a>. The offensive use of <a href="https://nim-lang.org/" target="_blank">Nim</a> is really heating up and this post walks through taking a template for C# loading in Nim and productizing it with Rubeus as well as using it to bypass Constrained Language mode for Powershell.</li>
<li><a href="https://adepts.of0x.cc/vba-outlook/" target="_blank">The worst of the two worlds: Excel meets Outlook</a>. Ever wanted to drop a VBA script from Excel that will be run by Outlook to keylog a target? Well now you can! The VBA tricks from <a href="https://twitter.com/TheXC3LL" target="_blank">@TheXC3LL</a> have been very impressive, and this is no exception.</li>
<li><a href="https://luemmelsec.github.io/Pentest-Everything-SMTP/" target="_blank">Pentest - Everything SMTP</a> demystifies the basics of SMTP and some simple commands you can use to try to spoof messages and enumerate users. Defenses against these techniques are also included.</li>
<li><a href="https://blog.scrt.ch/2020/12/28/state-of-pentesting-2020/" target="_blank">State of Pentesting 2020</a> gives a good overview of some common techniques and methods used on internal pentests and how to defend against them.</li>
<li><a href="https://arielkoren.com/blog/2020/12/24/forging-malicious-doc/" target="_blank">Forging malicious DOC, undetected by all VirusTotal static engines</a>. Breaking CRCs or inserting false headers cause all kinds of issues with parsers, and parsing is the first step most AV take with modern Office doucments.</li>
</ul>

<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/med0x2e/ExecuteAssembly" target="_blank">ExecuteAssembly</a> is one of the best tool drops in recent memory. ExecuteAssembly is an alternative of CS execute-assembly, built with C/C++ and it can be used to Load/Inject .NET assemblies by; reusing the host (spawnto) process loaded CLR Modules/AppDomainManager, Stomping Loader/.NET assembly PE DOS headers, Unlinking .NET related modules, bypassing ETW+AMSI, avoiding EDR hooks via NT static syscalls (x64) and hiding imports by dynamically resolving APIs via superfasthash hashing algorithm. This has all the building blocks to incorporate into your own tools or use directly in Cobalt Strike. The only thing missing from making this perfect is Hell's Gate for dynamically resolving the syscalls vs statically embedding them.</li>
<li><a href="https://github.com/sgabe/winaflpet" target="_blank">winaflpet</a> is a web user interface dedicated to WinAFL remote management via an agent running as a system service on fuzzing machines. The purpose of this project is to allow easy monitoring of fuzzing jobs running on several remote machines. This is a great project for anyone running a fuzzing farm against Windows software.</li>
<li><a href="https://github.com/redplait/pexphide" target="_blank">pexphide</a> is a PoC for hiding PE exports so that <cite>dumpbin /exports</cite> shows no exports for the DLL. No blog post or further information that I could find, so dig into the code for the details.</li>
<li><a href="https://github.com/mubix/solarflare" target="_blank">solarflare</a> is a credential dumping tool for SolarWinds Orion. With the recent news, now you can see what the attackers may have had access to in your environment. Lots more details an a full conference talk <a href="https://malicious.link/post/2020/solarflare-release-password-dumper-for-solarwinds-orion/" target="_blank">here</a></li>
<li><a href="https://github.com/slaeryan/DetectCobaltStomp" target="_blank">DetectCobaltStomp</a> when module stomping is enabled with <cite>module_x86/x64</cite> in a malleable C2 profile, this tool can detect the remnants left in memory. Also works against <a href="https://offensivedefence.co.uk/posts/module-stomping/" target="_blank">Module Stomping in C#</a>.</li>
<li><a href="https://github.com/jfmaes/SharpZipRunner" target="_blank">SharpZipRunner</a> executes position independent shellcode from an encrypted zip on disk. Modify this to ether pull the zip file via HTTPS or embed it and keep it all in memory for extra stealth.</li>
<li><a href="https://github.com/MaximeBeasse/KeyDecoder" target="_blank">KeyDecoder</a> is a mobile app (Android only currently) created for pentesters and security enthusiasts. From the picture of a key, one can measure its bitting easily. The picture must be of the target key on an ISO/CEI 7810 ID-1 sized card like a credit card for the computer vision magic to work properly. Available on <a href="https://play.google.com/store/apps/details?id=com.keydecoder" target="_blank">Google Play</a>.</li>
<li><a href="https://github.com/ant4g0nist/Vulnerable-Kext" target="_blank">Vulnerable-Kext</a> is a work-in progress "Vulnerable by Design" kext for iOS/macOS to play/learn with *OS kernel exploitation.</li>
<li><a href="https://github.com/certego/PcapMonkey" target="_blank">PcapMonkey</a> is a project that will provide an easy way to analyze pcap using the latest version of Suricata and Zeek. It can also save Suricata and Zeek logs in Elasticsearch using the new Elasticsearch Common Schema or the original field names. More details <a href="https://www.certego.net/en/news/introducing-pcapmonkey/" target="_blank">here</a>.</li>
<li><a href="https://halove23.blogspot.com/2020/12/oh-so-you-have-antivirus-nameevery-bug.html" target="_blank">Oh, so you have an antivirus... name every bug</a>. Some :fire: Windows local privilege escalation exploits in this post. Proof of concepts available <a href="https://github.com/klinix5/blog-stuff" target="_blank">on GitHub</a>.</li>
<li><a href="https://github.com/rvrsh3ll/NoMSBuild" target="_blank">NoMSBuild</a> uses the <cite>Microsoft.Build.Evaluation</cite> C# namespace to load and build a project without executing MSbuild.exe. Comes with a sample project that uses D/Invoke to inject shellcode.</li>
<li><dl>
<dt>Azure Reporting Tools</dt>
<dd><ul>
<li><a href="https://github.com/CrowdStrike/CRT" target="_blank">CRT</a> is the CrowdStrike Reporting Tool for Azure. This tool queries the following configurations in the Azure AD/O365 tenant which can shed light on hard to find permissions and configuration settings in order to assist organizations in securing these environments.</li>
<li><a href="https://github.com/cisagov/Sparrow" target="_blank">Sparrow</a> was created by CISA's Cloud Forensics team to help detect possible compromised accounts and applications in the Azure/m365 environment. The tool is intended for use by incident responders, and focuses on the narrow scope of user and application activity endemic to identity and authentication based attacks seen recently in multiple sectors.</li>
</ul>
</dd>
</dl>
</li>
<li><a href="https://bugs.chromium.org/p/project-zero/issues/detail?id=2096" target="_blank">Issue 2096: Windows: splWOW64 Elevation of Privilege</a>. This is an 0day (might be patched 2021-01-12?) that is a new way to exploit the same bug as <a href="https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2020-0986" target="_blank">CVE-2020-0986</a>. Only a basic PoC exists thus far (a proof of write-what-where), but if you are in need of such a primitive for Windows, today is your lucky day!</li>
<li><a href="https://github.com/thalpius/ADFS-Info" target="_blank">ADFS-Info</a> is a small project to get the private key and token-signing certificate from an AD FS server to create forged tokens that <a href="https://thalpius.com/2020/12/28/microsoft-defender-for-identity-adfsdump/" target="_blank">won't be detected by  Microsoft Defender for Identity</a>.</li>
<li><a href="https://github.com/scannells/exploits/tree/master/CVE-2020-27194" target="_blank">CVE-2020-27194</a>. The proof of concept exploit for CVE-2020-27194 (Ubuntu 20.10 eBPF local privilege escalation) has been released. Write up <a href="https://scannell.me/fuzzing-for-ebpf-jit-bugs-in-the-linux-kernel/" target="_blank">here</a>.</li>
</ul>

<section id="new-to-me">
<h2>New to Me</h2>
<p>This section is for news, techniques, and tools that weren't released last week but are new to me. Perhaps you missed them too!</p>
<ul>
<li><a href="https://github.com/t94j0/log" target="_blank">log</a> is a simple command line log book for offensive operations. It makes it easy to timestamp important events and screenshots and organize them by target.</li>
<li><a href="https://github.com/zackelia/ghidra-dark" target="_blank">ghidra-dark</a> provides a simple to use installer to install the FlatLaf dark theme, custom colors for disassembly/decompilation in Ghidra, and some other helpful settings. The installer supports all public builds through version 9.2 on Windows, Linux, and macOS.</li>
<li><a href="https://justine.lol/ape.html" target="_blank">αcτµαlly pδrταblε εxεcµταblε</a> is truly some black magic that allows gcc to take in pure C to output a binary that can run on every Linux distro in addition to macOS, Windows, FreeBSD, and OpenBSD too. The implications for a platform independent stager are obvious. More information <a href="https://justine.lol/cosmopolitan/index.html" target="_blank">here</a>.</li>
<li><a href="https://github.com/rafael-santiago/aegis" target="_blank">aegis</a> is a library that allows you detect if your software is being debugged or not on Linux, FreeBSD, NetBSD, OpenBSD and Windows. You can use it natively from C or use the Go bind.</li>
<li><a href="https://github.com/Orange-Cyberdefense/arsenal" target="_blank">arsenal</a> is just a quick inventory, reminder and launcher for pentest commands. This project written by pentesters for pentesters simplify the use of all the hard-to-remember commands. Reminds me of <a href="https://github.com/dbrgn/tealdeer" target="_blank">tealdeer</a> for pentesters.</li>
</ul>
