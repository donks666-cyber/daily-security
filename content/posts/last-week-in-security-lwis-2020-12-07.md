<p>Last Week in Security is a summary of the interesting cybersecurity news, techniques, tools and exploits from the previous week. This post covers 2020-11-30 to 2020-12-07.</p>
<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://www.zdnet.com/article/kazakhstan-government-is-intercepting-https-traffic-in-its-capital/" target="_blank">Kazakhstan government is intercepting HTTPS traffic in its capital</a>. Sound familiar? This is the 3rd attempt by Kazakhstan to pull this off. I hope browser makers block the TLS certificate like they did in the past, but the fact this is being tried at all is worrying. I would be very interested to see if <a href="https://github.com/SixGenInc/Noctilucent" target="_blank">Noctilucent</a> or <a href="https://github.com/iyouport-org/relaybaton" target="_blank">relaybaton</a> are effective or if ESNI is blocked to force the use of the man-in-the-middle'd certificate.</li>
<li><a href="https://aws.amazon.com/blogs/aws/new-use-mac-instances-to-build-test-macos-ios-ipados-tvos-and-watchos-apps/" target="_blank">New – Use Amazon EC2 Mac Instances to Build &amp; Test macOS, iOS, ipadOS, tvOS, and watchOS Apps</a>. This is great for devs, but since they are dedicated machines, there is a minimum of tenancy of 24 hours and cost $1.083 per hour. M1 chip based macs are in the works but not available yet. If this interests you, <a href="https://www.macstadium.com/pricing" target="_blank">MacStadium</a> is much cheaper for anything over 54 hours a month.</li>
<li><a href="https://googleprojectzero.blogspot.com/2020/12/an-ios-zero-click-radio-proximity.html" target="_blank">An iOS zero-click radio proximity exploit odyssey</a>. I don't really know how to classify this one. It's news, techniques, and an exploit all in one. A remote, unauthenticated remote code execution vulnerability in Apple's AirDrop protocol was not only found but meaningfully exploited by a single researcher (<a href="https://twitter.com/i41nbeer" target="_blank">@i41nbeer</a>) working alone. This is what being at the pinnacle of exploit development looks like. More great iOS research was released this week: <a href="https://bellis1000.medium.com/aslr-the-ios-kernel-how-virtual-address-spaces-are-randomised-d76d14dc7ebb" target="_blank">ASLR &amp; the iOS Kernel</a>, <a href="https://github.com/seemoo-lab/toothpicker" target="_blank">toothpicker</a> an in-process, coverage-guided fuzzer for iOS, and <a href="https://www.synacktiv.com//en/publications/ios-1-day-hunting-uncovering-and-exploiting-cve-2020-27950-kernel-memory-leak.html" target="_blank">iOS 1-day hunting: uncovering and exploiting CVE-2020-27950 kernel memory leak</a>.</li>
<li><a href="https://arxiv.org/pdf/2012.00687.pdf" target="_blank">Hey Alexa what did I just type? Decoding smartphone sounds with a voice assistant</a>. This is scary research that shows the ability to identify key patterns from sound is not limited to physical keyboards, but also works against virtual keyboards on smartphones and tablets.</li>
<li><a href="https://krebsonsecurity.com/2020/12/account-hijacking-site-ogusers-hacked-again/" target="_blank">Account Hijacking Site OGUsers Hacked, Again</a>. Once again a site built by hackers is hacked, this time via an out of date plugin. The hacker is offering to remove users from an upcoming leak for a fee.</li>
</ul>
</section>
<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://www.forrest-orr.net/post/a-modern-exploration-of-windows-memory-corruption-exploits-part-i-stack-overflows" target="_blank">A Modern Exploration of Windows Memory Corruption Exploits - Part I: Stack Overflows</a>. Forrest Orr (<a href="https://twitter.com/_ForrestOrr" target="_blank">@_ForrestOrr</a>) puts together a great primer on stack overflow exploits, and unlike nearly every other resource, he uses Windows as the target OS which brings its own set of challenges. This is a great resource for any aspiring exploit developer. Source code to follow along is <a href="https://github.com/forrest-orr/ExploitDev" target="_blank">here</a>.</li>
<li><a href="https://github.com/pedrib/PoC/blob/master/advisories/Micro_Focus/Micro_Focus_OBM.md" target="_blank">Multiple (RCE) Vulnerabilities in Micro Focus Operations Bridge Manager</a>. Who watches the watchers? OBM is a product to monitor and identify IT infrastructure problems, but comes with its own set of security issues - including the classic Java deserialization.</li>
<li><a href="https://labs.nettitude.com/blog/cve-2020-14418-madcodehook-library-local-privilege-escalation/" target="_blank">CVE-2020-14418: madCodeHook Library Local Privilege Escalation</a>. 'Tis the season of Windows local privilege escalation vulnerabilities in drivers. The Cisco ExPrevDriver.sys module, formally known as MorphiDriver32.sys has a time-of-check-time-of-use (TOCTOU) vulnerabilities that allows for arbitrary DLL loading.</li>
<li><a href="https://search.unprotect.it/map/" target="_blank">The Unprotect Project</a> is a MITRE ATT&amp;CK like collection of specific, technical techniques malware uses from <a href="https://twitter.com/fr0gger_" target="_blank">@fr0gger_</a> and <a href="https://twitter.com/darkcodersc" target="_blank">@DarkCoderSc</a>. Excited to see where this goes!</li>
<li><a href="https://mp.weixin.qq.com/s/fhcTTWV4Ddz4h9KxHVRcnw" target="_blank">Bypass cobaltstrike beacon config scan</a>. This post discusses the recent (last week's LWiS) Cobalt Strike beacon configuration scanners/parsers that connect to a Cobalt Strike that is configured for staged payloads (default) and dumps their configuration. By reversing Cobalt Strike itself, Zero Team was able to modify the static stage encryption key (a single byte XOR) and repack Cobalt Strike. Blue teams should brute force this key (256 possibilities) when attempting to parse a staged payload from a suspected Cobalt Strike server.</li>
<li><a href="https://shogunlab.gitbook.io/building-c2-implants-in-cpp-a-primer/" target="_blank">Building C2 Implants in C++: A Primer</a> goes through the process of designing a C2 infrastucture, listening post, implant, tasking, and a CLI client.</li>
<li><a href="https://offensivedefence.co.uk/posts/ppidspoof-blockdlls-dinvoke/" target="_blank">D/Invokify PPID Spoofy &amp; BlockDLLs</a>. RastaMouse keeps the hits coming with this post on D/Invoke (direct syscalls) and PPID spoofing. This should help anyone developing their own C# tools to bypass API hooks and some AV detection.</li>
<li><a href="https://www.secjuice.com/find-us-court-and-property-records-osint/" target="_blank">How to Find Someone's U.S. Court and Property Records</a>. If your target(s) are in the US, this post has lots of good resources to potentially uncover great phishing pretext.</li>
<li><a href="https://labs.sentinelone.com/apt32-multi-stage-macos-trojan-innovates-on-crimeware-scripting-technique/" target="_blank">APT32 Multi-stage macOS Trojan Innovates on Crimeware Scripting Technique</a>. There are some interesting, and some sloppy, techniques in this post. The initial execution and file swap is a great trick that is likely to be effective against many macOS users. Consider implementing it on your next macOS targeted phishing engagement.</li>
<li><a href="https://blog.syscall.party/post/weaponizing-windows-sandbox/" target="_blank">Weaponizing Windows Sandbox To Bypass Defender</a>. This is an interesting technique that uses the Windows Sandbox to hide malicious activity. Malware has done similar things by shipping an entire VM and Virtualbox to a target to run ransomware without detection, but the built in Windows Sandbox is in place on Windows 10 Pro and Enterprise, and can be launched with a .wsb file (phishing potential?).</li>
<li><a href="https://www.fortinet.com/blog/threat-research/leaking-browser-url-protocol-handlers" target="_blank">Leaking Browser URL/Protocol Handlers</a>. Fortinet has some good research (and a CVE) on fingerprinting installed applications using URL/protocol handlers, aka shortlinks. Different responses to these shortlinks indicated if an application was installed or not. The vendor response is along party lines: Mozilla issues a CVE and quickly fixes, Microsoft says it's not a security issue, and Google basically says they'll get around to it when they get a chance.</li>
</ul>
</section>
<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/blacklanternsecurity/writehat" target="_blank">writehat</a> is a reporting tool which removes Microsoft Word from the reporting process. Markdown --&gt; HTML --&gt; PDF. Created by penetration testers, for penetration testers - but can be used to generate any kind of report. Written in Django (Python 3). This and <a href="https://github.com/GhostManager/Ghostwriter/" target="_blank">Ghostwriter</a> (also written with Django) are interesting options for reporting.</li>
<li><a href="https://github.com/securesean/DecryptAutoLogon" target="_blank">DecryptAutoLogon</a> is a command line tool (C#) to extract/decrypt the password that was stored in the LSA by SysInternals AutoLogon.</li>
<li><a href="https://github.com/cube0x0/SharpMapExec" target="_blank">SharpMapExec</a> is a C# version of <a href="https://github.com/byt3bl33d3r/CrackMapExec" target="_blank">CrackMapExec</a> and is able to both scan for access, identify vulnerable configurations, and exfiltrate data using Kerberos or NTLM authentication. CME is one of my favorite Windows tools, so a C# version is very exciting.</li>
<li><a href="https://gist.github.com/slaeryan/2c73c4c4e33dfd7d8ce38312aacc9324" target="_blank">GetOSVersion.h</a> is a useful function for developing direct syscall solutions that uses the kuser data structure to pull version information, avoiding any API calls before using the correct direct syscalls.</li>
<li><a href="http://www.mac4n6.com/blog/2020/12/3/apollo-v14-now-with-gather-function-from-iosmacos-and-updates-to-ios14-and-macos-11-modules" target="_blank">APOLLO v1.4 - Now with 'Gather' Function from iOS/macOS and updates to iOS14 and macOS 11 modules</a>. Apple Pattern Of Life Lazy Outputter (APOLLO) is a great tool for macOS and iOS forensics, and it just got a big update for the latest operating systems.</li>
<li><a href="https://github.com/nccgroup/Carnivore" target="_blank">Carnivore</a> is a tool for assessing on-premises Microsoft servers such as ADFS, Skype, Exchange, and RDWeb. NCC group has a <a href="https://research.nccgroup.com/2020/12/03/tool-release-carnivore-microsoft-external-assessment-tool/" target="_blank">blog post</a> about the tool, as well as a <a href="https://www.youtube.com/watch?v=LJFuydx-HtE" target="_blank">DEF CON 28 presentation</a> (<a href="https://media.defcon.org/DEF%20CON%2028/DEF%20CON%20Safe%20Mode%20demo%20labs/DEF%20CON%20Safe%20Mode%20-%20Demo%20Labs%20-%20Chris%20Nevin%20-%20Carnivore.pdf" target="_blank">slides</a>).</li>
<li><a href="https://github.com/beurtschipper/Depix" target="_blank">Depix</a> is a tool for recovering passwords from pixelized screenshots. Use black boxes to redact and ensure they are not removable objects!</li>
<li><a href="https://github.com/tech-srl/Nero" target="_blank">Nero</a>. Now this is a cool application of machine learning. "Neural Reverse Engineering of Stripped Binaries using Augmented Control Flow Graphs" is the title of the <a href="https://arxiv.org/pdf/1902.09122.pdf" target="_blank">paper</a>. Who wants to implement this as a Ghidra plugin?!</li>
<li><a href="https://github.com/leonjza/tc2" target="_blank">tc2</a> is a repo of example C2 stacks, fronted using Traefik to avoid <a href="https://engineering.salesforce.com/easily-identify-malicious-servers-on-the-internet-with-jarm-e095edac525a" target="_blank">JARM</a> signature detection.</li>
<li><a href="https://github.com/jrozner/slipstream" target="_blank">slipstream</a> is a Go implementation of the NAT Slipstream attack from 2020-11-02's LWiS. The benefit is a single binary for client and server to check if your router is vulnerable.</li>
<li><a href="https://github.com/rabbitstack/fibratus" target="_blank">fibratus</a> is a modern tool for the Windows kernel exploration and tracing written in Go. It uses Event Tracing for Windows (ETW), so if that is being bypassed, this tool will be unable to pick it up. This could be a good addition to a blue team toolkit.</li>
<li><a href="https://github.com/ret2jazzy/ELFPatch" target="_blank">ELFPatch</a> is an x86/64 Python library to manipulate and patch ELFs with dynamically sized patches. This could be super useful for exploit development.</li>
<li><a href="https://github.com/projectdiscovery/cloudlist" target="_blank">cloudlist</a> is a multi-cloud tool for getting Assets (Hostnames, IP Addresses) from Cloud Providers. This is intended to be used by the blue team to augment Attack Surface Management efforts by maintaining a centralized list of assets across multiple clouds with very little configuration efforts. This tool requires an access token for each cloud service to enumerate the assets, making it much more of a blue team tool than red.</li>
<li><a href="https://github.com/S3cur3Th1sSh1t/SharpLigolo" target="_blank">SharpLigolo</a> is a C# wrapper of the Go tool <a href="https://github.com/sysdream/ligolo" target="_blank">ligolo</a> which is a simple and lightweight tool for establishing SOCKS5 or TCP tunnels from a reverse connection using TLS certificate for security. It is comparable to Meterpreter with Autoroute + Socks4a, but more stable and faster, and now wrapped in C#!</li>
<li><a href="https://github.com/S3cur3Th1sSh1t/Sharp-HackBrowserData" target="_blank">Sharp-HackBrowserData</a> is a C# wrapper of the Go tool <a href="https://github.com/moonD4rk/HackBrowserData" target="_blank">HackBrowserData</a> which can decrypt passwords, bookmarks, cookies and history from every browser except Internet Explorer and Safari across all three major OSs.</li>
</ul>
</section>
<section id="new-to-me">
<h2>New to Me</h2>
<p>This section is for news, techniques, and tools that weren't released last week but are new to me. Perhaps you missed them too!</p>
<ul>
<li><a href="https://github.com/phiresky/ripgrep-all" target="_blank">ripgrep-all</a> is a line-oriented search tool that allows you to look for a regex in a multitude of file types. rga wraps the awesome ripgrep and enables it to search in pdf, docx, sqlite, jpg, movie subtitles (mkv, mp4), etc. Useful for everyday use, or to find credentials and other interesting data on a target machine.</li>
<li><a href="https://github.com/WerWolv/ImHex" target="_blank">ImHex</a> is a Hex Editor for Reverse Engineers, Programmers and people that value their eye sight when working at 3 AM.</li>
<li><a href="https://github.com/tenzir/vast" target="_blank">vast</a>, or Visibility Across Space and Time, is a network telemetry engine for data-driven security investigations with sub-second response times over the entire zeek/suricata/json/csv imported data lake, thanks to multi-level bitmap indexing and actor model concurrency.</li>
</ul>
