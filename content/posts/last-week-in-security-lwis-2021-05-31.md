<p>Last Week in Security is a summary of the interesting cybersecurity news, techniques, tools and exploits from the previous week. This post covers 2021-05-24 to 2021-05-31.</p>
<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://blogs.vmware.com/vsphere/2021/05/vmsa-2021-0010.html" target="_blank">VMSA-2021-0010: What You Need to Know</a>. "There is a remote code execution vulnerability in the vSAN plugin, which ships as part of vCenter Server. This vulnerability can be used by anyone who can reach vCenter Server over the network to gain access, regardless of whether you use vSAN or not." No PoC yet, but considering vCenter/ESXi usually host most critical IT assets, this should be an emergency patch. Get the NSE checker <a href="https://github.com/alt3kx/CVE-2021-21985_PoC" target="_blank">here</a>, and more analysis <a class="m-text m-dim" href="https://attackerkb.com/topics/X85GKjaVER/cve-2021-21985#rapid7-analysis" target="_blank">here</a>.</li>
<li><a href="https://blog.cloudflare.com/quic-version-1-is-live-on-cloudflare/" target="_blank">QUIC Version 1 is live on Cloudflare</a>. QUIC is a new transport protocol built on top of UDP. Unfortunately, to access HTTP resources with HTTP/3, QUIC still includes an unencrypted Client Hello which includes the server name. Can you combine Encrypted Client Hello and QUIC (add ECH support to the <a href="https://github.com/cloudflare/quiche/blob/master/tools/apps/src/client.rs" target="_blank">test client</a> and let me know)? QUIC also has some interesting things in the pipeline (<a href="https://datatracker.ietf.org/wg/masque/about/" target="_blank">masque</a>).</li>
<li><a href="https://security.googleblog.com/2021/05/introducing-half-double-new-hammering.html" target="_blank">Introducing Half-Double: New hammering technique for DRAM Rowhammer bug</a>. Attackers can now use a second controlled row, which has a "non-liner gating effect" to "transport" the Rowhammer effect from the first to the row, through the second, to a victim row and is an intrinsic property of the underlying silicon substrate. No practical implications yet, but now you know when the CISO asks if you need to remediate for the new row hammer variant. JEDEC has published two documents about DRAM and system-level mitigation techniques (<a href="https://www.jedec.org/standards-documents/docs/jep300-1" target="_blank">JEP300-1</a> and <a href="https://www.jedec.org/standards-documents/docs/jep301-1" target="_blank">JEP301-1</a>).</li>
<li><a href="https://arstechnica.com/gadgets/2021/05/amazon-devices-will-soon-automatically-share-your-internet-with-neighbors/" target="_blank">Amazon devices will soon automatically share your Internet with neighbors</a>. Many Amazon devices will start allowing other amazon devices owned by different customers to use their internet connection to connect back to Amazon. If you don't want to be part of the world's largest botnet, opt-out now or don't buy IoT devices that are thinly veiled tools of surveillance.</li>
<li><a href="https://lockboxx.blogspot.com/2021/05/book-release-adversarial-tradecraft-in.html" target="_blank">Book Release: "Adversarial Tradecraft in Cybersecurity"</a>. Dan's book drops June 9th, and looks to be a great collection of techniques and tricks for both red and blue teams. This will be a must read for any CCDC or ProsVJoes competitors!</li>
<li><a href="https://m1racles.com/" target="_blank">M1ssing Register Access Controls Leak EL0 State</a>. A register on the M1 chip can be read and written to by any process, enabling two pieces of malware on the same machine to communicate. Big deal? Probably not (there are much easier ways for malware to communicate on the same system), but it could <a href="https://github.com/pavel-kirienko/cpu-load-side-channel" target="_blank">allow malware in separate virtual machines to communicate</a>.</li>
<li><a href="https://www.wired.com/story/the-full-story-of-the-stunning-rsa-hack-can-finally-be-told/" target="_blank">The Full Story of the Stunning RSA Hack Can Finally Be Told</a>. Before "supply chain attack" was a buzzword, the Chinese were harvesting RSA seeds straight from the source. Despite the dramatic language, the operation wasn't all that novel. Phish a user, expand access, harvest data.</li>
</ul>
</section>
<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://www.jamf.com/blog/zero-day-tcc-bypass-discovered-in-xcsset-malware/" target="_blank">Zero-Day TCC bypass discovered in XCSSET malware</a>. Dropping a malicious .app inside the folder of an app that was given screen recording permissions was enough to bypass Apple's protections and allow a malicious actor to take screenshots without the user approving the malware to do so. This was patched in macOS 11.4. PoC that captures 15 seconds of camera without asking if placed inside an app directory that has approved camera access (facetime, photobooth, etc): <a href="https://github.com/theevilbit/macos/tree/master/CaptureCam" target="_blank">CaptureCam</a>.</li>
<li><a href="https://bohops.com/2021/05/30/abusing-and-detecting-lolbin-usage-of-net-development-mode-features/" target="_blank">Abusing and Detecting LOLBIN Usage of .NET Development Mode Features</a>. With the right privileges, it is possible to modify the configuration for .Net binaries in system32 and (along with setting an environment variable) leverage them for application control bypass with managed assembly modification (depending on the solution), general DLL hijack/sideloading, and persistence.</li>
<li><a href="https://www.gosecure.net/blog/2021/05/27/step-by-step-how-to-deanonymize-emails-on-linkedin/" target="_blank">Step-by-step how to deanonymize emails on LinkedIn</a>. With the Outlook integration (thanks to the Microsoft purchase of LinkedIn), anyone with an Outlook account and an intercepting proxy can leak the emails of users on LinkedIn (pro tip: check your <a href="https://www.linkedin.com/help/linkedin/answer/83634" target="_blank">privacy settings</a>!).</li>
<li><a href="https://research.nccgroup.com/2021/05/27/detecting-rclone-an-effective-tool-for-exfiltration/" target="_blank">Detecting Rclone – An Effective Tool for Exfiltration</a>. Ransomware gangs often pull large amounts of target data before encrypting a network, and this post looks into ways to detect this exfiltration hopefully in time to stop the encryption.</li>
<li><a href="https://theevilbit.github.io/posts/teamviewer_lpe/" target="_blank">TeamViewer Local Privilege Escalation Vulnerability</a>. This is a writeup of a macOS privilege escalation in TeamViewer from 2020-11, but the technique may be applicable to other apps that run privileged helpers.</li>
<li><a href="https://www.synacktiv.com/en/publications/baking-mojolicious-cookies.html" target="_blank">Baking Mojolicious cookies</a>. Much like JWT, the Perl web framework Mojolicious cookies can be cracked if predictable weak keys are used.</li>
<li><a href="https://posts.specterops.io/the-attack-path-management-manifesto-3a3b117f5e5" target="_blank">The Attack Path Management Manifesto</a>. The team at SpecterOps are masters of the Attack Path and this manifesto lays out their thinking.</li>
<li><a href="https://v3ded.github.io/redteam/abusing-lnk-features-for-initial-access-and-persistence" target="_blank">Abusing LNK "Features" for Initial Access and Persistence</a>. LNK files can be set to run on a key shortcut and can be hidden. Writing an LNK file to the desktop from a macro and having the shortcut run a payload that injects a C2 tool and removes the LNK (restoring the shortcut) could gain stealthy initial access and the target would only notice one broken copy function. This has the added benefit of spawning your C2 loader as a child of explorer.exe naturally.</li>
</ul>

<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://straightblast.medium.com/my-poc-walkthrough-for-cve-2021-21974-a266bcad14b9" target="_blank">My RCE PoC walkthrough for (CVE-2021–21974) VMware ESXi OpenSLP heap-overflow vulnerability</a>. This is a detailed walk through of the development process of a heap-overflow for ESXi and the <a href="https://github.com/straightblast/My-PoC-Exploits/blob/master/CVE-2021-21974.py" target="_blank">first public PoC</a> for it (that I am aware of).</li>
<li><a href="https://github.com/gentilkiwi/mimikatz/releases" target="_blank">2.2.0 20210525 MSTSC Passwords</a> is a new Mimikatz release that can dump plaintext RDP credentials without injection or a hooking. This technique was discussed in LWiS 2021-05-17.</li>
<li><a href="https://github.com/memflow/scanflow" target="_blank">scanflow</a> boasts a feature set similar to the likes of CheatEngine, with a simple command line interface. Utilizing memflow, scanflow works in a wide range of situations - from virtual machines, to dedicated DMA hardware.</li>
<li><a href="https://github.com/thalpius/Microsoft-JSON-Web-Token-Extractor" target="_blank">Microsoft-JSON-Web-Token-Extractor</a> is a C# project to extract JSON Web Tokens from memory without dumping anything on disk to avoid detection by Endpoint Detection and Response. Check out the <a href="https://thalpius.com/2021/05/27/microsoft-json-web-token-extractor/" target="_blank">blog post</a> for more information.</li>
<li><a href="https://github.com/checkymander/Carbuncle" target="_blank">Carbuncle</a> is a tool for interacting with outlook interop during red team engagements. Not a new release, but 0.2 adds a lot of nice features. Be warned, depending on the setup this could cause a popup to be created for the user!</li>
<li><a href="https://github.com/xforcered/InvisibilityCloak" target="_blank">InvisibilityCloak</a> is a proof-of-concept obfuscation toolkit for C# post-exploitation tools. For more information check out the <a href="https://securityintelligence.com/posts/invisibility-cloak-obfuscate-c-tools-evade-signature-based-detection/" target="_blank">blog post</a>.</li>
<li><a href="https://github.com/GetRektBoy724/SharpUnhooker" target="_blank">SharpUnhooker</a> is a C# universal API unhooker - automatically unhook functions from ntdll.dll, kernel32.dll, user32.dll, and kernelbase.dll.</li>
<li><a href="https://github.com/aaaddress1/wowInjector" target="_blank">wowInjector</a> is a PoC to exploit the 32-bit thread snapshot of WOW64 to take over $RIP, inject &amp; bypass AV.</li>
<li><a href="https://github.com/STMSolutions/boobsnail" target="_blank">boobsnail</a> allows generating Excel 4.0 XLM macro. Its purpose is to support the RedTeam and BlueTeam in XLM macro generation.</li>
<li><a href="https://github.com/its-a-feature/loginItemManipulator" target="_blank">loginItemManipulator</a> is an Objective-C tool to list, add, and remove login items for user's session and global loginitem lists for macOS.</li>
</ul>
