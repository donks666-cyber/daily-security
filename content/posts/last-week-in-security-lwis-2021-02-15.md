<h2>News</h2>
<ul>
<li><a href="https://www.securityweek.com/remote-hacker-caught-poisoning-florida-city-water-supply" target="_blank">Remote Hacker Caught Poisoning Florida City Water Supply</a>. This has happened <a href="https://www.securityweek.com/iranian-hackers-access-unprotected-ics-israeli-water-facility" target="_blank">before</a>, but the quote that got me was, "The Sheriff said the remote access itself wasn't unusual." Why is remote access to a water treatment plant normal? Even if for some reason it is justified, TeamViewer is the solution they settled on!?</li>
<li><a href="https://msrc-blog.microsoft.com/2021/02/09/multiple-security-updates-affecting-tcp-ip/" target="_blank">Multiple Security Updates Affecting TCP/IP: CVE-2021-24074, CVE-2021-24094, and CVE-2021-24086</a>. IPv4 source routing and IPv6 packet reassembly on Windows can be exploited to gain remote code execution. It's too early to tell how these vulnerabilities will play out (i.e. are they actually exploitable/reliable?) but Microsoft went out of their way to highlight them in a blog post. Expect to see these CVE numbers pop up throughout 2021 as researchers dig into the patch diffs.</li>
<li><a href="https://www.bleepingcomputer.com/news/security/trickbots-bazarbackdoor-malware-is-now-coded-in-nim-to-evade-antivirus/" target="_blank">TrickBot's BazarBackdoor malware is now coded in Nim to evade antivirus</a>. If you are a regular reader of LWiS you saw this coming. Does this make LWiS "threat intelligence" now?!</li>
<li><a href="https://twitter.com/CDPROJEKTRED/status/1359048125403590660" target="_blank">Cyberpunk 2077 developers held to ransom after cyber-attack, source code theft</a>. In a bonus twist, the source code has reportedly <a href="https://twitter.com/vxunderground/status/1359864618659819520" target="_blank">been sold</a>.</li>
<li><a href="https://www.bloomberg.com/features/2021-supermicro/" target="_blank">The Long Hack: How China Exploited a U.S. Tech Supplier</a>. Remember <a href="https://www.bloomberg.com/news/features/2018-10-04/the-big-hack-how-china-used-a-tiny-chip-to-infiltrate-america-s-top-companies" target="_blank">the 2018 story</a> that made some very big claims with very little evidence? It's back, this time with more sources including one on the record. Despite the article doubling down on its claims that "thousands" of servers were compromised and 10's of private companies targeted, we have yet to see any physical evidence. You would think that in the last 3 years at least one such chip would have been found and shown to the public?</li>
</ul>

<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://medium.com/@alex.birsan/dependency-confusion-4a5d60fec610" target="_blank">Dependency Confusion: How I Hacked Into Apple, Microsoft and Dozens of Other Companies</a>. "From one-off mistakes made by developers on their own machines, to misconfigured internal or cloud-based build servers, to systemically vulnerable development pipelines, one thing was clear: squatting valid internal package names was a nearly sure-fire method to get into the networks of some of the biggest tech companies out there, gaining remote code execution, and possibly allowing attackers to add backdoors during builds." It was only a matter of time before this was exploited on a wide scale. 2021 is shaping up to be the "year of the software supply chain." You can use <a href="https://github.com/visma-prodsec/confused" target="_blank">Confused</a> to find free namespaces for private package names in your own projects, on assessments, or in bug bounty eligible projects.</li>
<li><a href="https://swarm.ptsecurity.com/swarm-of-palo-alto-pan-os-vulnerabilities/" target="_blank">Swarm of Palo Alto PAN-OS vulnerabilities</a>. The authenticated RCEs show some creative ways to inject commands via user forms and XML as well as analysis of how/why they work.</li>
<li><a href="https://labs.sentinelone.com/cve-2021-24092-12-years-in-hiding-a-privilege-escalation-vulnerability-in-windows-defender/" target="_blank">CVE-2021-24092: 12 Years in Hiding – A Privilege Escalation Vulnerability in Windows Defender</a>. Hard link based local privilege escalation vulnerabilities are still showing up in Windows after quite some time of hard scrutiny. How many more are left to find?</li>
<li><a href="https://halove23.blogspot.com/2021/02/windows-installer-file-read-0day_12.html" target="_blank">Windows Installer File Read 0day</a>. Using a symlink and legitimate MSI installer, the Windows install process can be abused to allow arbitrary file reads. PoC <a href="https://github.com/klinix5/Windows_Installer_File_Read" target="_blank">here</a>.</li>
<li><a href="https://adepts.of0x.cc/hookson-hootoff/" target="_blank">Hooks-On Hoot-Off: Vitaminizing MiniDump</a>. When a minidump of lsass was successful but the resulting file was deleted, <a href="https://twitter.com/Mari0Bartolome" target="_blank">@Mari0Bartolome</a> uses a function hook to exfiltrate the file to a waiting server instead of writing it to disk. For a more in-spec implementation of this, see the second section of: <a href="https://www.ired.team/offensive-security/credential-access-and-credential-dumping/dumping-lsass-passwords-without-mimikatz-minidumpwritedump-av-signature-bypass" target="_blank">Dumping Lsass without Mimikatz with MiniDumpWriteDump</a>. Want it in C#? <a href="https://github.com/m0rv4i/SafetyDump" target="_blank">I've got you covered</a>.</li>
<li><a href="https://www.crummie5.club/the-lone-sharepoint/" target="_blank">The Lone Sharepoint</a>. You may think a Sharepoint server that requires authentication and does not allow outbound traffic (besides related/established HTTPS) would be fairly secure. <a href="https://twitter.com/acap4z" target="_blank">@acap4z</a> shows how some CVEs, a little knowhow, and a lot of persistence debugging and re-writing webshell proxies can allow such a machine to a gateway into a network. Some proper hacking in this post.</li>
<li><a href="https://blog.syscall.party/post/windows-10-stealthy-threads/" target="_blank">Stealthy Process Communication Between Threads on Windows 10</a>. While pipes are the standard method of interprocess communication (IPC) on Windows, <a href="https://twitter.com/LloydLabs" target="_blank">@LloydLabs</a> has developed a PoC using ThreadName to communicate between processes without pipes. Add this to your advanced red team implant to keep pipe-aware EDR/AV off your tail. Mudge has a timely article about pipes in Cobalt Strike if you are curious to learn more: <a href="https://blog.cobaltstrike.com/2021/02/09/learn-pipe-fitting-for-all-of-your-offense-projects/" target="_blank">Learn Pipe Fitting for all of your Offense Projects</a>.</li>
<li><a href="https://winternl.com/detecting-manual-syscalls-from-user-mode/?utm_source=rss&amp;utm_medium=rss&amp;utm_campaign=detecting-manual-syscalls-from-user-mode" target="_blank">Detecting Manual Syscalls from User Mode</a>. As direct syscalls become more popular, defenders will slowly adapt methods to detect them. One method is to check if the instruction pointer of a kernel to user mode return is in ntdll.dll or wind32u.dll (the legitimate sources of syscalls into the kernel) and alerting if not. This can be done from user mode with a built in framework to instrument all kernel to user mode callbacks called Nirvana. There are good ideas for future work and bypasses at the end of the post and the code is available <a href="https://github.com/jackullrich/syscall-detect" target="_blank">here</a>. Top notch research.</li>
<li><a href="https://medium.com/@markmotig/bypass-av-edr-with-safe-mode-975aacecc809" target="_blank">Bypass AV/EDR with Safe Mode?</a>. If you can afford to reboot a machine, you may be able to defeat AV/EDR by booting into safe mode!</li>
</ul>

<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/FuzzySecurity/Dendrobate" target="_blank">Dendrobate</a> is a framework that facilitates the development of payloads that hook unmanaged code through managed .NET code. To do this Dendrobate uses EasyHook and packs the injected component, Dendron, using Fody &amp; Costura. This is all done automatically so all you need to do when you compile Dendron is select the appropriate .NET version (only v3.5 and v4.5 are supported!). Don't sleep on the full MFA bypass walkthrough in the second half of the readme, it could be a whole blog post on its own.</li>
<li><a href="https://github.com/itm4n/Perfusion" target="_blank">Perfusion</a> is a "forever-day" (won't be patched) local privilege escalation for Windows 7, 8, Server 2008R2, and 2012. The registry key of the RpcEptMapper and DnsCache (7/2008R2 only) services is configured with weak permissions. Any local user can create a Performance subkey and then leverage the Windows Performance Counters to load an arbitrary DLL in the context of the WMI service as NT AUTHORITYSYSTEM (hence the tool's name). More information in a <a href="https://itm4n.github.io/windows-registry-rpceptmapper-eop/" target="_blank">blog post</a> from last November.</li>
<li><a href="https://github.com/doronz88/hilda" target="_blank">hilda</a> is a debugger which combines both the power of LLDB and iPython for easier debugging on a jailbroken iPhone.</li>
<li><a href="https://github.com/ModernPwner/cicuta_virosa" target="_blank">cicuta_virosa</a> is an iOSiPadOS 14.3 kernel LPE for all devices by @ModernPwner that exploits CVE-2021-1782: A race condition in user_data_get_value() leading to ivac entry uaf. This issue has been actively exploited in the wild with a WebKit exploit. The RCE chain may be released in the future.</li>
<li><a href="https://github.com/ajpc500/BOFs/tree/main/StaticSyscallsAPCSpawn" target="_blank">StaticSyscallsAPCSpawn</a> Executes shellcode on Windows using: NtCreateSection -&gt; NtMapViewOfSection -&gt; NtQueueApcThread -&gt; NtResumeThead. It also uses BeaconSpawnTemporaryProcess to create the target process. You may want to edit the NtCreateSection calls for better opsec (RW local, RX remote).</li>
<li><a href="https://github.com/rookuu/BOFs/tree/main/MiniDumpWriteDump" target="_blank">MiniDumpWriteDump</a> is a custom implementation of DbgHelp's MiniDumpWriteDump function. It uses static syscalls to replace low-level functions like NtReadVirtualMemory. Since this is a full custom dumper plus static syscalls, it likely has a long shelf life.</li>
<li><a href="https://github.com/conwnet/github1s" target="_blank">github1s</a> - Just add 1s after GitHub in a URL and press Enter in the browser address bar for any repository you want to read in a web version of VSCode. GitHub1s implements a VS Code Extension (includes FileSystemProvider) that uses GitHub's REST API as a filesystem, then serves this as a static site. Very clever!</li>
<li><a href="https://github.com/EgeBalci/WSAAcceptBackdoor" target="_blank">WSAAcceptBackdoor</a> is a POC implementation for a DLL implant that acts as a backdoor for accept Winsock API calls. Once the DLL is injected into the target process, every accept call is intercepted using the Microsoft's detour library and redirected into the BackdooredAccept function. When a socket connection with a pre-defined special source port is established, BackdooredAccept function launches a cmd.exe process and binds the accepted socket to the process STD(OUT/IN) using a named pipe.</li>
<li><a href="https://github.com/aus/gopherheaven" target="_blank">gopherheaven</a> is a Go implementation of the classic <a href="https://vx-underground.org/archive/VxHeaven/lib/vrg02.html" target="_blank">Heaven's Gate technique</a> originally published by roy g biv on VX Heaven in 2009. gopherheaven can be used as an evasion technique to directly call 64-bit code from a 32-bit process. Looking for Hell's Gate instead? That has been around for a little while as <a href="https://github.com/C-Sto/BananaPhone" target="_blank">BananaPhone</a>.</li>
</ul>
