<p>Last Week in Security is a summary of the interesting cybersecurity news, techniques, tools and exploits from the previous week. This post covers 2021-01-18 to 2021-01-25.</p>
<section id="news">
<h2>News</h2>
<ul>
<li><a href="https://www.sonicwall.com/support/product-notification/urgent-security-notice-probable-sma-100-series-vulnerability-updated-jan-25-2021/210122173415410/" target="_blank">Urgent Security Notice: Probable SMA 100 Series Vulnerability</a>. SonicWall, and its customers, had a bad weekend after the hardware VPN vendor announced there was a probably 0day being actively exploited in the wild and the only mitigation was to allowlist known good clients. Today the update the statement to indicate that besides the NetExtender product, all SonicWall products may be affected. To add insult to injury, <a href="https://twitter.com/_darrenmartyn" target="_blank">@_darrenmartyn</a> dropped the "0day" (patched in 2015) that Phineas Fisher used to completely dismantle <a href="https://rmusser.net/docs/Rants&amp;Writeups/Hacking%20Team%20Writeup.html" target="_blank">Hacking Team</a> and <a href="https://theanarchistlibrary.org/library/subcowmandante-marcos-hack-back" target="_blank">Cayman Trust Bank</a>. The code is <a href="https://github.com/darrenmartyn/VisualDoor" target="_blank">on github</a> and exploits the ancient shellshock vulnerability.</li>
<li><a href="https://www.raspberrypi.org/blog/raspberry-pi-silicon-pico-now-on-sale/" target="_blank">Meet Raspberry Silicon: Raspberry Pi Pico now on sale at $4</a>. Raspberry Pi drops their own custom microcontroller chip, a dual core Cortex-M0+ with 264kb of RAM. They also brought in partners to help with software and support <a href="https://micropython.org/" target="_blank">MicroPython</a> and <a href="https://github.com/raspberrypi/pico-tflmicro" target="_blank">TensorFlow Lite Micro</a>. Full datasheet <a href="https://datasheets.raspberrypi.org/pico/pico_datasheet.pdf" target="_blank">here</a>, and SDK documentation is available as well (<a href="https://datasheets.raspberrypi.org/pico/sdk/pico_c_sdk.pdf" target="_blank">C/C++</a>, and <a href="https://datasheets.raspberrypi.org/pico/sdk/pico_python_sdk.pdf" target="_blank">Python</a>). Want other languages? It won't be long; <a href="https://github.com/rp-rs/pico-blink-rs" target="_blank">Rust has already been ported</a> and more are sure to follow.</li>
<li><a href="https://blog.snort.org/2021/01/snort-3-officially-released.html" target="_blank">Snort 3 officially released</a>. This release brings claims of greater speeds, more efficient rules, a new plugin system, and more. <a href="https://blog.snort.org/2020/08/snort-3-2-differences.html" target="_blank">This post</a> has a breakdown of the differences between Snort 2 and Snort 3.</li>
<li><a href="https://www.microsoft.com/security/blog/2021/01/20/deep-dive-into-the-solorigate-second-stage-activation-from-sunburst-to-teardrop-and-raindrop/" target="_blank">Deep dive into the Solorigate second-stage activation: From SUNBURST to TEARDROP and Raindrop</a>. Microsoft details the link between the Solar Winds backdoor and multiple Cobalt Strike loaders used for hands-on-keyboard exploitation activities. The use of "commodity" red team tools after such a well executed backdooring is a bit strange. Either one group handed off access to a less advanced group for "effects" on targets or perhaps the threat actors hoped the discovery of Cobalt Strike would not trigger an in depth incident response that would burn the Solar Winds backdoor.</li>
<li><a href="https://blog.google/threat-analysis-group/new-campaign-targeting-security-researchers/" target="_blank">New campaign targeting security researchers</a>. This one feels close to home. A threat actor has been baiting security researchers with fake PoCs (I even linked one of the fake YouTube demos last week) and getting them to open malicious Visual Studio projects that infect their systems when built. Even more scary, there are reports of fully patched Windows 10 and Chrome visitors to the honeypot "exploit research blog" getting infected as well. If you read this blog, it might be worth checking your machine for the IOCs at the end of this article.</li>
<li><a href="https://twitter.com/cmwdotme/status/1351838924621099008" target="_blank">Linux is now completely usable on the Mac mini M1</a>. This is a huge step, and perhaps one day all the performance optimizations in the M1 will be available to Linux. Code <a href="https://github.com/corellium/linux-m1" target="_blank">here</a>. There is also the <a href="https://asahilinux.org/" target="_blank">Asashi Linux</a> project which is aiming to make Linux usable on Apple Silicon.</li>
<li><dl>
<dt>Linux Lockscreen Bypass News</dt>
<dd><ul>
<li><a href="https://github.com/linuxmint/cinnamon-screensaver/issues/354" target="_blank">Screensaver lock by-pass via the virtual keyboard</a>. Kids playing on a locked Linux Mint desktop manage to crash the screensaver and gain access. Epic.</li>
<li><a href="https://twitter.com/jcase/status/1353393704891899904" target="_blank">Ubuntu budgie lock screen bypass</a>. Adding and removing monitors can cause issues with screen locks, or in this case crash them all together.</li>
</ul>
</dd>
</dl>
</li>
</ul>

<section id="techniques">
<h2>Techniques</h2>
<ul>
<li><a href="https://ajpc500.github.io/nim/Shellcode-Injection-using-Nim-and-Syscalls/" target="_blank">Shellcode Injection using Nim and Syscalls</a>. <a href="https://nim-lang.org/" target="_blank">Nim</a> is the latest language to be embraced by the offensive security community, and as it gains popularity more advanced and low level techniques are being ported to Nim. This post uses <a href="https://github.com/ajpc500/NimlineWhispers" target="_blank">NimlineWhispers</a> to enable direct syscalls on Windows in Nim.</li>
<li><a href="https://medium.com/csis-techblog/silencing-microsoft-defender-for-endpoint-using-firewall-rules-3839a8bf8d18" target="_blank">Silencing Microsoft Defender for Endpoint using firewall rules</a>. As more enterprises enable Windows Defender for Endpoint (aka Windows Defender ATP) red teams will need to check for and adjust their post-exploitation activity accordingly. If an attacker has local admin privileges, firewall rules can selectively block the host from communicating with Microsoft cloud endpoints responsible for the advanced reporting in Security Center. Like any other security solution, lack of data should be an indicator.</li>
<li><a href="https://blog.gigamon.com/2021/01/20/dns-c2-sandwich-a-novel-approach/" target="_blank">DNS C2 Sandwich: A Novel Approach</a>. By sending a base64 encoded query sandwiched between two legitimate queries in a DNS request, you can fool some (zeek and Suricata &lt; 6) network monitoring tools.</li>
<li><a href="https://wojciechregula.blog/post/when-vulnerable-library-is-actually-your-physical-book/" target="_blank">When vulnerable library is actually your physical book</a></li>
<li><a href="https://john-woodman.com/research/malicious-vba-macros-trials-tribulations/" target="_blank">Malicious VBA Macro’s: Trials and Tribulations</a>. VBA/macros remain a useful phishing tool and this post discusses how to call the Windows API via VBA, the issues involved, and how to get a pure VBA reverse shell and memory dumper down to zero Virus Total detections. While the reverse shell or memory dumper may not be immediately useful, the Windows API use and obfuscation techniques can be applied to other VBA projects.</li>
<li><a href="https://www.crowdstrike.com/blog/cve-2021-1678-printer-spooler-relay-security-advisory/?utm_content=sprout&amp;utm_medium=soc&amp;utm_source=twittere.com/blog/cve-2021-1678-printer-spooler-relay-security-advisory/" target="_blank">Security Advisory: MSRPC Printer Spooler Relay (CVE-2021-1678)</a>. The Windows print spooler is the gift that <a href="https://vulners.com/metasploit/MSF:EXPLOIT/WINDOWS/SMB/MS10_061_SPOOLSS" target="_blank">keeps</a> <a href="https://github.com/leechristensen/SpoolSample" target="_blank">on</a> <a href="https://windows-internals.com/printdemon-cve-2020-1048/" target="_blank">giving</a>. This time, the IRemoteWinSpool MSRPC interface is abused in combination with NTLM relaying to install a malicious printer driver and "print" a document to it. A PoC based on impacket is coming soon.</li>
<li><a href="https://research.nccgroup.com/2021/01/21/mssql-lateral-movement/" target="_blank">MSSQL Lateral Movement</a>. While most MSSQL techniques require <cite>xp_cmdshell</cite>, this one loads the CLR and then a .NET executable remotely. A tool to help generate a standalone shellcode execution executable with all the requirements (connection string, SHA512 hash, etc) is available <a href="https://github.com/nccgroup/nccfsas/tree/main/Tools/Squeak" target="_blank">here</a>.</li>
<li><a href="https://labs.nettitude.com/blog/vm-detection-tricks-part-1-physical-memory-resource-maps/" target="_blank">VM Detection Tricks, Part 1: Physical memory resource maps</a>. This is some cool research that eschews the typical methods of VM detection (cpu cores, memory size, drivers, etc) and instead goes for something much harder to spoof, and much less signatured - Physical memory resource maps. This is a great potential detection you should consider implementing in your next red team tool.</li>
<li><a href="https://offensivedefence.co.uk/posts/dinvoke-syscalls/" target="_blank">Syscalls with D/Invoke</a>. Syscalls are hot. D/Invoke is hot. The combination is even hotter.</li>
<li><a href="https://googleprojectzero.blogspot.com/2021/01/windows-exploitation-tricks-trapping.html" target="_blank">Windows Exploitation Tricks: Trapping Virtual Memory Access</a>. James Forshaw is one of the grandmasters of unique Windows exploits. This one is low level, but worth the read if you are doing Windows exploit development.</li>
<li><a href="https://secret.club/2021/01/20/diet-process.html" target="_blank">Process on a diet: anti-debug using job objects</a>. This one is a bit limiting, as it requires small programs with limited dynamic allocations to work properly, but the use of built in Windows systems to prevent debugging is always fun.</li>
<li><a href="https://blog.shoebpatel.com/2021/01/23/The-Secret-Parameter-LFR-and-Potential-RCE-in-NodeJS-Apps/" target="_blank">The Secret Parameter, LFR, and Potential RCE in NodeJS Apps</a>. This one is for the web app testers and bug bounty hunters out there. The magic/horror of JavaScript allows for some strange behavior and potentially and RCE (if the version of handlebars is old enough and you can upload a template file to then include as a layout). This post reminds me of <a href="https://youtu.be/RPvORV2Amic?t=1140" target="_blank">one of my favorite security conference talks</a>.</li>
</ul>
</section>
<section id="tools-and-exploits">
<h2>Tools and Exploits</h2>
<ul>
<li><a href="https://github.com/klinix5/Appx_Blog/tree/main/APPX_LPE_2" target="_blank">Appx_Blog</a> is a PoC for a Windows LPE that exists if a drive has multiple partitions. Not very common, but may be useful as it is a 0day. Demo <a href="https://youtu.be/zEcNwEHGI9g" target="_blank">here</a>; there is a <a href="https://halove23.blogspot.com/2021/01/another-privilege-escalation-in-windows.html" target="_blank">short blog post</a> as well.</li>
<li><a href="https://github.com/mhaskar/bughound" target="_blank">Bughound</a> is a static code analysis tool for Java and PHP based on Elasticsearch. Details <a href="https://shells.systems/unveiling-bughound-a-static-code-analysis-tool-based-on-elasticsearch/" target="_blank">here</a>.</li>
<li><a href="https://github.com/pwn1sher/CS-BOFs/tree/main/lsass" target="_blank">lsass</a> is a Cobalt Strike Beacon Object File to dump lsass memory but uses a snapshot of lsass memory to do the minidump which may help evade some AV/EDR products.</li>
<li><a href="https://github.com/cedowens/JXA-RemoveQuarantine" target="_blank">JXA-RemoveQuarantine</a> is a JXA script based on research by Jeff Johnson on leveraging TextEdit to remove quarantine attributes on files. It does create a brief pop up on macOS &gt; 10.13.</li>
<li><a href="https://github.com/LloydLabs/delete-self-poc" target="_blank">delete-self-poc</a>  is a PoC that shows a way to delete a locked file, or current running executable, on disk in Windows. This is a common tactic on Linux, but it was previously impossible to delete a running binary on Windows.</li>
<li><a href="https://github.com/JamesCooteUK/SharpSphere" target="_blank">SharpSphere</a> gives red teamers the ability to easily interact with the guest operating systems of virtual machines managed by vCenter. Full blog post <a href="https://jamescoote.co.uk/introducing-sharpsphere/" target="_blank">here</a>.</li>
</ul>
</section>
